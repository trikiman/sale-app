# Plan 02: Admin Dashboard Proxy Health Section

<objective>
Add a proxy pool health section to the admin dashboard showing current pool status, recent failures, and success rates. Uses existing `/admin/proxy-history` and `/admin/proxy-status` endpoints — no backend changes needed.
</objective>

## Tasks

<task id="02.1">
<title>Add proxy health section to admin dashboard</title>
<read_first>
- backend/admin.html (look for existing sections pattern — scraper status cards, etc.)
- backend/main.py (lines 3584-3596 — `/admin/proxy-history` endpoint returns `{periods: {today, week, month}, recent: [...]}`)
</read_first>
<action>
Add a "Proxy Pool" section to `backend/admin.html` after the existing scraper status section. The section should:

1. **Pool Status Card**: Show current pool size, last refresh time, number of healthy proxies
2. **Stats Table**: Show today/week/month stats from `/admin/proxy-history`:
   - Found (new working proxies discovered)
   - Removed (dead proxies evicted)
   - Success rate %
   - Refreshes (pool refresh cycles)
3. **Recent Events**: Last 10 events with timestamp, type (found/removed/test_fail), and proxy addr

Fetch data from existing endpoints:
- `GET /admin/proxy-history` (with X-Admin-Token header) → returns periods + recent events

Use the same CSS patterns as existing admin sections (cards, tables, status badges). Add auto-refresh every 60 seconds for the proxy section.

The section heading should be "🔄 Proxy Pool" to match the emoji style of other admin sections.
</action>
<acceptance_criteria>
- backend/admin.html contains "Proxy Pool" section heading
- backend/admin.html fetches `/admin/proxy-history` endpoint
- backend/admin.html displays today/week/month stats in a table
- backend/admin.html displays recent proxy events list
- Section auto-refreshes (setInterval with 60000ms or similar)
</acceptance_criteria>
</task>
