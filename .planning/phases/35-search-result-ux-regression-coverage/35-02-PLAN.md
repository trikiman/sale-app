# Plan 02: Search Result State Regression Tests

## Objective

Add lightweight frontend regression coverage so the new mixed-result labels and state classification do not silently regress after future History page changes.

## must_haves

- [ ] Frontend tests exercise live-sale, history-only, and catalog-only classification
- [ ] Frontend tests assert the user-facing label/support-copy contract for those states
- [ ] CSS tests confirm the new state-chip styles exist in `index.css`
- [ ] Coverage stays inside the repo’s existing `node:test` pattern
