# Plan 02: Product Catalog Backfill And Merge Regression Coverage

## Objective

Extend local catalog seeding/backfill so newly discovered products become searchable locally and add tests for merge precedence and product_catalog population.

## must_haves

- [ ] Newly discovered products flow into `product_catalog`
- [ ] Existing richer metadata is preserved
- [ ] Discovery images backfill into `product_catalog`
- [ ] Merge/backfill rules are covered by tests

## Tasks

<task id="01" title="Extend product_catalog backfill to include merged discovery rows and non-destructive metadata updates">
<read_first>
- database/sale_history.py
- data/category_db.json
- .planning/phases/37-catalog-merge-backfill/37-CONTEXT.md
- .planning/phases/37-catalog-merge-backfill/37-RESEARCH.md
</read_first>

<action>
Update `database/sale_history.py:seed_product_catalog()` so it backfills newly discovered products into `product_catalog` safely.

1. Read discovery-backed fields from `category_db.json`, including `image_url` when present.
2. For new products, insert rows into `product_catalog` even if category/group/subgroup are blank.
3. For existing products, fill only missing fields such as blank `image_url`; do not overwrite richer existing taxonomy.
4. Keep the update path non-destructive and compatible with existing history stats.
</action>

<acceptance_criteria>
- `seed_product_catalog()` reads discovery-backed fields from `category_db.json`
- newly discovered products can be inserted into `product_catalog` with blank taxonomy fields
- existing non-empty taxonomy fields are preserved
- `image_url` can backfill into `product_catalog`
</acceptance_criteria>
</task>
