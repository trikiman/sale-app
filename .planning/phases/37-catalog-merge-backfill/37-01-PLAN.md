# Plan 01: Merged Discovery Artifact And Category DB Backfill

## Objective

Create a deduped merged discovery artifact from the per-source files and backfill `category_db.json` additively, preserving richer existing metadata while adding newly discovered products.

## must_haves

- [ ] All source files merge into one deduped discovery artifact
- [ ] Existing richer taxonomy in `category_db.json` is preserved
- [ ] Newly discovered products are added to `category_db.json`
- [ ] Source provenance is retained in merged discovery or category data

## Tasks

<read_first>

<action>
Create `merge_catalog_discovery.py` and use it to merge the Phase 36 source files into one deduped discovery artifact, then update `data/category_db.json` additively.

1. Read `data/catalog_discovery_state.json` and only include stable/non-blocking sources allowed by the Phase 36 verification contract.
2. Read all eligible `data/catalog_discovery_sources/*.json` files and merge them by numeric `product_id`.
3. Write a deduped merged artifact containing:
   - `product_id`
   - `name`
   - `url`
   - `image_url`
   - `source_slugs`
   - `source_names`
4. Update `data/category_db.json`:
   - if a product already exists, preserve non-empty `category`, `group`, and `subgroups`
   - fill only blanks or additive provenance fields
   - if a product is new, add a row with at least `name`, `image_url` when available, and discovery source provenance
5. Do not overwrite richer existing taxonomy with weaker overlay-source labels.
</action>

<acceptance_criteria>
- `merge_catalog_discovery.py` exists
- `merge_catalog_discovery.py` writes a deduped merged discovery artifact
- `merge_catalog_discovery.py` updates `data/category_db.json`
- existing non-empty `group` / `subgroups` values in `category_db.json` are preserved
- newly discovered products are added to `category_db.json`
</acceptance_criteria>
</task>
