# Plan 01: Dual-Cadence Scheduler

## Objective

Replace the single fixed scheduler interval with a due-job scheduler that keeps full cycles on a 5-minute target and opportunistically runs green-only refreshes on a 1-minute target between them.

## must_haves

- [ ] Full `ALL` cycles target every 5 minutes
- [ ] Extra `GREEN-only` cycles target every 1 minute
- [ ] No overlapping scraper runs
- [ ] Full `ALL` cycle wins if another green run would make red/yellow late

## Tasks

<task id="01" title="Replace the single interval loop with explicit due-job scheduling">
<read_first>
- scheduler_service.py
- .planning/phases/40-freshness-aware-scheduler-alerts/40-CONTEXT.md
</read_first>

<action>
Refactor `scheduler_service.py` so the main loop schedules two job types: `all_cycle` and `green_only`.

1. Replace the single `INTERVAL_MINUTES = 3` wait model with:
   - `ALL_TARGET_SECONDS = 300`
   - `GREEN_TARGET_SECONDS = 60`
2. Track `next_all_due_at` and `next_green_due_at` using monotonic time.
3. Implement a green-only execution path that runs:
   - `scrape_green.py`
   - `scrape_merge.py`
   - `backend/notifier.py`
4. After each green-only run, set `next_green_due_at = run_started_at + GREEN_TARGET_SECONDS`.
5. After each full cycle, set:
   - `next_all_due_at = run_started_at + ALL_TARGET_SECONDS`
   - `next_green_due_at = now + GREEN_TARGET_SECONDS`
6. If both jobs are due, or if starting another green run would make the next full cycle late, run the full cycle and skip that extra green pass.
7. Keep the existing no-overlap, retry, and proxy behavior intact.
</action>

<acceptance_criteria>
- scheduler_service.py defines separate 300-second full-cycle and 60-second green-only targets
- scheduler_service.py has a green-only execution path that runs green, merge, and notifier
- scheduler_service.py no longer relies on one fixed `INTERVAL_MINUTES` sleep for all work
- full cycles are chosen over extra green runs when timing conflicts
</acceptance_criteria>
</task>
