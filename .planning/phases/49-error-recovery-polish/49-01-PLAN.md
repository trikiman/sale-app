# Plan 49-01: Error Type Mapping, Distinct Messages & Retry

<objective>
Parse `error_type` from backend cart-add responses, show distinct Russian error messages per failure mode, add retry-on-tap for retryable errors, and trigger login prompt for auth_expired — all without backend changes.
</objective>

<must_haves>
- Each backend error_type maps to a distinct user-facing message
- auth_expired triggers login prompt (not a toast)
- Retryable errors show replay icon and remain tappable for 4s
- Non-retryable errors show ❌ and reset after 2s
- ProductDetail mirrors the same retry/error behavior
</must_haves>

<action>
In `App.jsx` `handleAddToCart`, replace the error handling block (lines 976-1006) that currently does:
- 401 check → setShowLogin
- string-match sold-out → toast "Этот продукт уже раскупили"
- everything else → toast "Корзина временно недоступна"
- all errors → 2000ms reset

With error_type-based routing:

```javascript
// After: const data = await res.json()
// Replace the else block (non-success, non-pending path):

const errorType = data?.error_type || ''

if (errorType === 'auth_expired' || res.status === 401) {
  setIsAuthenticated(false)
  setShowLogin(true)
  setCartStates(s => ({ ...s, [pid]: null }))
  return
}

const isRetryable = ['transient', 'timeout'].includes(errorType)
const soldOut = errorType === 'product_gone'

const messageMap = {
  product_gone: 'Этот продукт уже раскупили',
  transient: 'ВкусВилл временно недоступен',
  timeout: 'Корзина не ответила вовремя',
}
const toastText = messageMap[errorType] || 'Корзина временно недоступна'

setToastMessage({ text: toastText, type: 'error' })
setTimeout(() => setToastMessage(null), isRetryable ? 4000 : 3000)

if (soldOut) {
  setSoldOutIds(s => {
    const next = new Set([...s, pid])
    try {
      localStorage.setItem('soldOutIds', JSON.stringify([...next]))
      localStorage.setItem('soldOutIds_expiry', String(Date.now() + 4 * 60 * 60 * 1000))
    } catch { }
    return next
  })
}

setCartStates(s => ({ ...s, [pid]: isRetryable ? 'retry' : 'error' }))
setTimeout(() => setCartStates(s => ({ ...s, [pid]: null })), isRetryable ? 4000 : 2000)
```

Also update the `catch` block (lines 1007-1024):
- AbortError: set cart state to `'retry'` (retryable), keep message "Корзина не ответила вовремя", use 4000ms timers
- Network error: set cart state to `'retry'` (retryable), keep message "Ошибка сети", use 4000ms timers

Also update the poll-exhausted handler (line 885): set cart state to `'retry'` instead of `'error'`, use 4000ms timer.

Also update poll error handler (lines 862-870): parse `error_type` from poll failure data, use same `isRetryable` logic, set `'retry'` state for retryable.
</action>
