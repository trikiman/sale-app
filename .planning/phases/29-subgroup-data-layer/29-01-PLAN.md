# Plan 01: Subgroup Data Layer

## Objective

Extend the category scraper to discover and scrape VkusVill subgroups, update the data schema to store `{group, subgroup}` per product, add `group`/`subgroup` columns to the `product_catalog` DB table, and flow the data through the merge pipeline and API responses.

## must_haves

- [ ] Category scraper discovers subgroup links from each category page sidebar
- [ ] Category scraper fetches each subgroup's product pages and maps products to subgroups
- [ ] `category_db.json` stores `{name, group, subgroups: [...]}` per product
- [ ] `product_catalog` DB table has `group` and `subgroup` TEXT columns
- [ ] Merge pipeline passes group/subgroup into proposals.json products
- [ ] History API returns `group` and `subgroup` fields per product
- [ ] Products API returns available groups with their subgroups list

## Tasks

<task id="01" title="Update category scraper to discover and scrape subgroups">
<read_first>
- scrape_categories.py (current scraper implementation)
- config.py (CATEGORY_GROUPS mapping)
</read_first>

<action>
Modify `scrape_categories.py` to:

1. Add a new async function `discover_subgroups(session, sem, category_url)`:
   - Fetch the main category page HTML
   - Parse the sidebar navigation links with CSS selector for subgroup links
   - VkusVill subgroup links are inside the category sidebar, pattern: `/goods/{category-slug}/{subgroup-slug}/`
   - Filter out "Все товары категории" (that's the main category page itself)
   - Return list of `{name: "Салаты", url: "https://vkusvill.ru/goods/gotovaya-eda/salaty/"}`

2. Modify `scrape_category()` to call `discover_subgroups()` first, then:
   - If subgroups found: scrape each subgroup page with pagination (same `_parse_products` logic)
   - Tag each product with `subgroup = subgroup_name`
   - Still scrape the main category page to catch products not in any subgroup
   - Products found in the main category page but NOT in any subgroup get `subgroup = None`

3. Update return type from `(category_name, [products])` to `(category_name, [products_with_subgroup])` where each product is `{id, name, subgroup}`.

4. Update `scrape_all_categories_async()` merge logic:
   - Change DB schema from `{id: {name, category}}` to `{id: {name, group, subgroups: []}}`
   - Where `group` = top-level category name (e.g., "Готовая еда")
   - Where `subgroups` = list of subgroup names the product appears in (e.g., ["Салаты", "Постные и веганские блюда"])
   - For products not in any subgroup: `subgroups: []`
   - When a product already exists in DB and is seen in a new subgroup, append to subgroups array (not first-write-wins for subgroups)
   - Group assignment uses first-write-wins (same as current category behavior)

5. CSS selector for subgroup links on VkusVill:
   The sidebar links are in the page navigation. From the HTML analysis, subgroup links follow this URL pattern:
   - Main category: `/goods/gotovaya-eda/`
   - Subgroup: `/goods/gotovaya-eda/salaty/` (3 path segments under /goods/)
   
   Parse all `<a>` tags with href matching `{category_url}{subgroup-slug}/` pattern.
   Exclude links containing "novinki" — those are special "Новинки" marketing pages.
   
   Wait — per user decision D-04, VkusVill's "Новинки" IS a real subgroup. Keep it.

6. Handle categories with no subgroups gracefully:
   - Some categories (e.g., "Мороженое") may have no subgroup links
   - In that case, just scrape all products from the main category as before
   - All products get `subgroup = None`, `group = category_name`
</action>

<task id="02" title="Add group/subgroup columns to product_catalog table">
<read_first>

<action>
1. In `database/db.py`, update `_init_sale_history_tables()`:
   - Add `group TEXT` and `subgroup TEXT` columns to `product_catalog` CREATE TABLE
   - Add migration: `ALTER TABLE product_catalog ADD COLUMN group_name TEXT` (use group_name to avoid SQL keyword conflict)
   - Add migration: `ALTER TABLE product_catalog ADD COLUMN subgroup TEXT`
   - Wrap ALTER in try/except to handle "column already exists" gracefully
   - Add index: `CREATE INDEX IF NOT EXISTS idx_catalog_group ON product_catalog(group_name)`
   - Add index: `CREATE INDEX IF NOT EXISTS idx_catalog_subgroup ON product_catalog(subgroup)`

2. In `database/sale_history.py`, update `record_sale_appearances()`:
   - Load `category_db.json` to get group/subgroup data
   - When upserting into `product_catalog`, populate `group_name` and `subgroup` columns
   - For `subgroup`, store the first/primary subgroup from the array (or comma-separated if multiple)
   - For `group_name`, store the top-level group name

The column is named `group_name` (not `group`) because `GROUP` is a SQL reserved keyword.

<read_first>

<action>
1. In `history_get_products()` (backend/main.py):
   - Add `pc.group_name, pc.subgroup` to the SELECT query (line ~3337)
   - Include `group` and `subgroup` in the product dict response
   - Update the category filter support:
     - Accept `group` query parameter to filter by group_name
     - Accept `subgroup` query parameter to filter by subgroup

2. Update the categories aggregation query (lines 3434-3446):
   - Instead of just `SELECT DISTINCT category`, also return groups with their subgroups:
   ```sql
   SELECT DISTINCT group_name, subgroup, COUNT(*) as cnt
   FROM product_catalog
   WHERE group_name IS NOT NULL AND group_name != ''
   GROUP BY group_name, subgroup
   ORDER BY cnt DESC
   ```
   - Return structured data: `{groups: [{name: "Готовая еда", subgroups: ["Салаты", "Супы", ...], count: N}, ...]}`

3. Add a new API endpoint `GET /api/groups`:
   - Returns all available groups with their subgroup lists
   - Used by frontend to populate filter chips
   - Response: `{groups: [{name: "Готовая еда", subgroups: [{name: "Салаты", count: N}, ...], count: N}, ...]}`
   - Include groups that have products currently on sale (for main page) or all groups (for history page) based on a `?scope=active|all` parameter
</action>
