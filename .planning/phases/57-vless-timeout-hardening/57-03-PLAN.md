# 57-03: Restore Egress Geo Verification

**Target:** Fix symptom-level bug S5 from `../56-vless-proxy-migration/INSPECTION-2026-04-23.md`: the v1.16 PR #7 (`90d2e9a`) removed multi-provider geo verification entirely, violating the phase-56 plan decision D-05 ("always geo-verify exit IPs"). As a result, the admitted pool egresses from FI/DE/NL/FR/PL (per 56-VERIFICATION.md Step 2) despite being labeled with Russian flag emojis.

**Estimated effort:** 45-60 minutes (1-2 file edits + test updates).
**Risk:** Medium — re-enables a check that was deliberately removed; must verify the replacement logic doesn't shrink the pool below `MIN_HEALTHY = 7`.

**Prerequisite:** 57-01 committed (so xray config has policy/observatory). 57-02 is NOT a prerequisite for 57-03 — they can ship in either order, but this plan assumes both are in place when the deploy script runs.

---

## Scope

### In scope
- Extend `_probe_candidates_in_parallel` at `vless/manager.py:623-702` to do an egress-country verification probe AFTER the existing VkusVill probe succeeds
- Use the existing `XrayProcess.verify_egress` helper — do NOT write new HTTP client code
- Reject candidates whose egress is not `RU` (case-insensitive)
- Preserve the existing emoji pre-filter in `vless/sources.py:filter_ru_nodes` (it's a cheap first-pass, not redundant)
- Track egress country in `VlessNode.extra` for diagnostics
- Update tests to cover the new probe path

### Out of scope
- Any change to `vless/sources.py:filter_ru_nodes` signature or emoji-detection logic
- Any change to `XrayProcess.verify_egress` (works as-is)
- Re-introducing the multi-provider consensus geo resolver during parsing (that was the v1.0-era approach; the new approach is probe-through-candidate)
- Caching egress results beyond pool-level `last_success_at` (keep it simple: re-verify on every refresh cycle)
- Changing `MIN_HEALTHY`, `MAX_CACHED`, or pool-size thresholds

## Implementation Steps

### Step 1 — Extend `_probe_one` in `vless/manager.py`

Read the current function at `vless/manager.py:632-674`. It currently:
1. Starts a per-candidate xray subprocess
2. Probes vkusvill.ru through it with `_probe_vkusvill`
3. If probe succeeds, returns the node with `probe_speed_s`
4. Otherwise returns None

Add a step 2.5: after `_probe_vkusvill` returns True, call `proc.verify_egress` and reject if country is not RU.

```python
def _probe_one(node: VlessNode, idx: int) -> VlessNode | None:
    if time.monotonic() > deadline:
        return None
    test_port = 20000 + (idx % 10_000)
    test_config = _BASE_DIR / "bin" / "xray" / "configs" / f"probe-{idx}.json"
    test_log = _BASE_DIR / "bin" / "xray" / "logs" / f"probe-{idx}.log"
    config_dict = build_xray_config([node], listen_port=test_port)
    try:
        test_config.parent.mkdir(parents=True, exist_ok=True)
        test_config.write_text(
            json.dumps(config_dict, indent=2, ensure_ascii=False),
            encoding="utf-8",
        )
    except OSError:
        return None

    proc = XrayProcess(
        binary=binary,
        config_path=test_config,
        log_path=test_log,
        health_check_timeout=4.0,
        restart_limit=0,
    )
    try:
        proc.start()
    except XrayStartupError:
        return None
    start = time.monotonic()
    try:
        vkusvill_ok = self._probe_vkusvill(
            proxy=f"{XRAY_LISTEN_HOST}:{test_port}",
            verify_ssl=False,
        )
        if not vkusvill_ok:
            return None
        # NEW: verify egress country. Restores plan-56 decision D-05 which
        # v1.16 PR #7 inadvertently dropped. See INSPECTION-2026-04-23.md S5.
        egress_ok, egress_country = proc.verify_egress(
            expected_country="RU",
            timeout=10.0,
            url="https://ipinfo.io/json",
        )
        if not egress_ok:
            node.extra["rejected_reason"] = f"egress_country={egress_country}"
            return None
        node.extra["egress_country"] = egress_country
        node.extra["probe_speed_s"] = round(time.monotonic() - start, 2)
        return node
    finally:
        proc.stop(timeout=3.0)
        try:
            test_config.unlink()
        except FileNotFoundError:
            pass
```

**Important:** move the return logic so the subprocess is always stopped (the existing `try/finally` already does this correctly; just make sure your added geo-check is inside the `try` block).

### Step 2 — Update `_probe_candidates_in_parallel` deadline math

The overall refresh deadline is `_REFRESH_MAX_TIME_S = 180` seconds. Adding a ~1-2 second geo probe per candidate increases per-candidate cost from ~8s to ~10s. With `_NODE_TEST_CONCURRENCY = 8`, 55 candidates take 55/8 * 10 ≈ 70s worst-case — still well under 180s.

No deadline change needed. Verify this holds by running the refresh in test mode:

```bash
pytest tests/test_vless_manager.py -v -k refresh
```

### Step 3 — Handle egress-probe failures gracefully

If `verify_egress` itself raises (network hiccup, ipinfo.io rate-limiting), treat it as "unknown country → reject." The existing `XrayProcess.verify_egress` already catches exceptions and returns `(False, "<error message>")`, so this is handled. Confirm by reading `vless/xray.py:235-261`.

### Step 4 — Track rejection stats

When a candidate is rejected due to non-RU egress, emit a structured log line so operators can diagnose pool-size shrink:

```python
if not egress_ok:
    self._log(f"Rejected {node.host} — egress_country={egress_country}")
    node.extra["rejected_reason"] = f"egress_country={egress_country}"
    return None
```

The log goes through `self._log` which writes to the operator log (stdout or Telegram via backend). This is a diagnostic, not a user-visible thing.

def test_probe_candidates_rejects_non_ru_egress(
    manager_bare, mock_xray_process, mocker

def test_probe_candidates_admits_ru_egress(
    manager_bare, mock_xray_process, mocker

    mock_xray_process.verify_egress = mocker.MagicMock(

    )
    admitted = manager_bare._probe_candidates_in_parallel([candidate])
    assert len(admitted) == 1
    assert admitted[0].extra.get("egress_country") == "RU"


def test_probe_candidates_rejects_when_vkusvill_fails_before_egress(
    manager_bare, mock_xray_process, mocker
):
    """If vkusvill probe fails, we should NOT waste time on egress probe."""
    manager_bare._probe_vkusvill = mocker.MagicMock(return_value=False)
    verify_egress_spy = mocker.spy(mock_xray_process, "verify_egress")

        uuid="cccc", host="10.0.0.1", port=443, name="fake",
        reality_pbk="PK", security="reality",
    )
    admitted = manager_bare._probe_candidates_in_parallel([candidate])
    assert len(admitted) == 0
    verify_egress_spy.assert_not_called()
