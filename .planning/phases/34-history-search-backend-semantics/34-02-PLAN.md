# Plan 02: Search Scope State Reset

## Objective

Align `HistoryPage.jsx` state transitions with the Phase 34 search contract so switching between history mode and active search does not silently preserve stale group/subgroup scope from the wrong dataset.

## must_haves

- [ ] Entering search mode clears stale history-scoped `selectedGroup` / `selectedSubgroup`
- [ ] Leaving search mode clears stale catalog-scoped `selectedGroup` / `selectedSubgroup`
- [ ] Group/subgroup selections made while search remains active are preserved
- [ ] Existing active-type/special-filter auto-clear behavior on typing search remains intact

## Tasks

<task id="01" title="Reset group and subgroup scope only when search mode changes">
<read_first>
- miniapp/src/HistoryPage.jsx
- .planning/phases/34-history-search-backend-semantics/34-CONTEXT.md
- .planning/ROADMAP.md
</read_first>

<action>
Modify `miniapp/src/HistoryPage.jsx` to make search-mode scope transitions intentional:

1. Track the boolean transition between `search === ''` and `search !== ''` using previous-state comparison.
2. When the page transitions from non-search mode into active search:
   - clear `selectedGroup`
   - clear `selectedSubgroup`
   - keep `groupsScope = 'all'`
3. When the page transitions from active search back to empty search:
   - clear `selectedGroup`
   - clear `selectedSubgroup`
   - allow the existing `groupsScope = 'history'` behavior to repopulate history-only chips
4. Do NOT clear `selectedGroup` or `selectedSubgroup` on every keystroke while `search` stays non-empty; if the user reselects a group/subgroup during active search, that choice must persist until the mode changes again.
5. Preserve the existing behavior where typing a search term clears stale sale-type and special filters (`activeTypes`, `specialFilter`) so search is not silently narrowed by them.
</action>

<acceptance_criteria>
- HistoryPage.jsx contains transition-aware logic for entering and leaving search mode
- entering search mode clears `selectedGroup` and `selectedSubgroup`
- leaving search mode clears `selectedGroup` and `selectedSubgroup`
- typing additional characters while search stays non-empty does not clear a group/subgroup the user selected during the active search session
- `groupsScope` still resolves to `search ? 'all' : 'history'`
</acceptance_criteria>
</task>
