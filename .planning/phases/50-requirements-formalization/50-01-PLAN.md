# Plan 50-01: Formalize Orphaned v1.13 Requirement IDs

## Goal

Define all 6 orphaned requirement IDs referenced in ROADMAP.md Phase 47-49 but missing from REQUIREMENTS.md. Add formal definitions with descriptions and update the traceability table.

## Context

The v1.13 milestone audit found that CART-15, CART-16, PERF-01, PERF-02, ERR-01, ERR-02 were referenced in ROADMAP.md but never formally defined in REQUIREMENTS.md. All implementations exist and are verified — this is a documentation gap only.

Replace the v1.12 REQUIREMENTS.md header and content with v1.13 definitions:

1. Update title to "Requirements — v1.13 Instant Cart & Reliability"
2. Add section "Cart Reliability (Phase 47)" with:
   - CART-15: Cart-add endpoint returns structured error_type field (auth_expired, product_gone, transient, timeout) instead of generic 500
   - CART-16: Backend logs show specific root cause for every cart-add failure with diagnostic context
3. Add section "Performance (Phase 48)" with:
   - PERF-01: On login, sessid and user_id are pre-extracted and cached so no warmup GET blocks the first cart add
   - PERF-02: Stale sessid (older than 30 min) is auto-refreshed before it causes a cart failure
4. Add section "Error Recovery (Phase 49)" with:
   - ERR-01: User sees distinct messages for sold-out, session-expired, VkusVill-down, and network-error states
   - ERR-02: After a transient error, user can retry the add without refreshing the page (retry state)
5. Mark all 6 as [x] (satisfied)
6. Update traceability table with all 6 IDs mapped to their implementing phases with "Satisfied" status
</action>
