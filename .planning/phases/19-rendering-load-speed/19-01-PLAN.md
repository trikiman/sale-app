# Plan 19-01: Backend API GZip Compression

<objective>
Add GZip compression middleware to the FastAPI backend so all API responses are automatically compressed. This reduces payload size by ~70% for JSON responses, significantly improving load times on mobile/tablet connections.
</objective>

## Tasks

<task id="1">
<title>Add GZipMiddleware to FastAPI</title>
<read_first>
- backend/main.py (lines 1-110 — imports and middleware setup)
- requirements.txt
</read_first>
<action>
Add GZipMiddleware from starlette to the FastAPI app. This is built into starlette (which FastAPI depends on) — no new dependencies needed.

1. Add import at the top of `backend/main.py`:
```python
from starlette.middleware.gzip import GZipMiddleware
```

2. Add middleware AFTER the CORS middleware (line ~104):
```python
app.add_middleware(GZipMiddleware, minimum_size=500)
```

The `minimum_size=500` means only responses >= 500 bytes get compressed. Small responses (health checks, single-item responses) skip compression overhead.

3. Verify `starlette` is already in requirements.txt (it's a FastAPI dependency). If not listed explicitly, that's fine — it's installed as part of FastAPI.
</action>
<acceptance_criteria>
- backend/main.py contains `from starlette.middleware.gzip import GZipMiddleware`
- backend/main.py contains `app.add_middleware(GZipMiddleware, minimum_size=500)`
- GZipMiddleware is added AFTER CORSMiddleware (order matters — CORS headers must be set before compression)
- Server starts without import errors: `python -c "from starlette.middleware.gzip import GZipMiddleware; print('OK')"` exits 0
</acceptance_criteria>
</task>
