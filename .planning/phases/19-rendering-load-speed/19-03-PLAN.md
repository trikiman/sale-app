# Plan 19-03: Tablet UX — Layout, Detail View & Image Optimization

<objective>
Make iPad/tablets get a mobile-optimized 2-column grid layout, convert the product detail drawer to a full-page view on tablets, and optimize image loading for mobile devices. Desktop remains completely unchanged.
</objective>

## Tasks

<task id="1">
<title>Add tablet-optimized grid layout via CSS</title>
<read_first>
- miniapp/src/index.css (search for grid layout rules — look for `grid-template-columns`, `.products-grid`, card layout rules)
- miniapp/src/App.jsx (search for `products-grid` or grid-related className)
</read_first>
<action>
Add CSS media queries to force a 2-column grid on tablets. This addresses the iPad rendering lag by reducing the number of columns and removing grid auto-sizing calculations.

Add at the END of `index.css` (after the backdrop-filter and reduced-motion blocks from Plan 02):

```css
/* === PERF-10: Tablet-optimized grid layout === */
@media (min-width: 481px) and (max-width: 1024px) {
  .products-grid {
    grid-template-columns: repeat(2, 1fr) !important;
    gap: 12px !important;
  }
}

/* Mobile stays 2-col (or 1-col on very small screens) */
@media (max-width: 480px) {
  .products-grid {
    grid-template-columns: repeat(2, 1fr) !important;
    gap: 8px !important;
  }
}
```

The exact class name for the product grid container must be verified by reading App.jsx and index.css. Find the grid container element and use its actual className.

IMPORTANT: Do NOT modify existing grid rules. Only add overrides via media queries at the end of the file.
</action>
<acceptance_criteria>
- index.css contains media query for `(min-width: 481px) and (max-width: 1024px)` with `grid-template-columns: repeat(2, 1fr)`
- The actual grid container class name from the codebase is used (not a guess)
- Desktop grid layout (> 1024px) is unchanged
- No existing grid CSS rules are modified
</acceptance_criteria>
</task>

<task id="2">

</read_first>
<action>
Make ProductDetail render as a full-page view on tablets instead of an overlay drawer. This avoids the janky overlay animation and backdrop-filter performance hit on iPads.

**Approach: CSS-only full-page override**

Add at the END of `index.css`:

```css
/* === PERF-11: Full-page product detail on tablets === */
@media (max-width: 1024px), (pointer: coarse) {
  .product-detail-overlay {
    /* Make overlay cover full screen without blur */
    position: fixed !important;
    inset: 0 !important;
    width: 100% !important;
    max-width: 100% !important;
    height: 100% !important;
    max-height: 100% !important;
    border-radius: 0 !important;
    transform: none !important;
    background: var(--bg-primary, #0a0a0f) !important;
    overflow-y: auto !important;
    -webkit-overflow-scrolling: touch;
  }

  .product-detail-overlay .detail-close-btn {
    position: sticky !important;
    top: 0;
    z-index: 10;
  }
}
```

The exact class names for the detail overlay/drawer must be verified by reading ProductDetail.jsx and index.css. Find the actual overlay container element and its close button, then update the selectors accordingly.

This is a CSS-only change — no JSX modifications needed. The drawer component still renders the same markup, but CSS forces it to full-screen on tablets. Desktop keeps the sliding drawer behavior.
</action>

<task id="3">
<title>Optimize image loading for mobile devices</title>
<read_first>
- miniapp/src/App.jsx (search for `proxyImg`, `product.image`, `img src` — understand current image loading)
- miniapp/src/App.jsx (lines 85-94 — ProductCard image rendering)
</read_first>
<action>
VkusVill CDN images are loaded at full resolution on all devices. Add CSS-based image optimization and check if VkusVill CDN supports image resizing via URL parameters.

1. **Check VkusVill CDN URL patterns.** Current image URLs look like `https://img.vkusvill.ru/...`. Check if appending width params (e.g., `?w=300`) returns smaller images.

2. **If CDN supports resize params:** Update the `proxyImg()` function in App.jsx to append size params on mobile:
```javascript
function proxyImg(url) {
  if (!url) return ''
  // On mobile/tablet, request smaller images
  if (window.innerWidth <= 1024 && url.includes('img.vkusvill.ru')) {
    // Append resize param if CDN supports it
    const separator = url.includes('?') ? '&' : '?'
    return url + separator + 'w=400'
  }
  return url
}
```

3. **If CDN does NOT support resize params (most likely):** Add CSS to constrain rendered image size and ensure hardware-accelerated compositing:
```css
/* === PERF-09: Image optimization on tablets/mobile === */
@media (max-width: 1024px), (pointer: coarse) {
  .card-hero-img {
    /* Ensure images don't decode at full resolution */
    image-rendering: auto;
    content-visibility: auto;
    contain: layout style paint;
  }

  .card-image-wrap {
    /* GPU-accelerated compositing for smooth scroll */
    will-change: transform;
    contain: layout style paint;
  }
}
```

4. **Add `decoding="async"` to all product images** in the `<img>` tag in ProductCard (App.jsx line ~86):
```jsx
<img
  src={proxyImg(product.image)}
  alt={product.name}
  referrerPolicy="no-referrer"
  loading="lazy"
  decoding="async"
  className={`card-hero-img ${imageLoaded ? 'loaded' : ''}`}
  onLoad={() => setImageLoaded(true)}
  onError={() => setImageError(true)}
/>
```

The `decoding="async"` attribute tells the browser to decode the image off the main thread, reducing jank during scrolling.
</action>
<acceptance_criteria>
- Product images in App.jsx have `decoding="async"` attribute
- index.css contains CSS containment rules for card images on tablets/mobile
- If VkusVill CDN supports resize: proxyImg() appends width param on mobile
- If CDN doesn't support resize: CSS containment and content-visibility applied instead
- Desktop image loading behavior is unchanged
- `loading="lazy"` is still present on all product images
</acceptance_criteria>
</task>
