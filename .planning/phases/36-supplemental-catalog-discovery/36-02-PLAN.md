# Plan 02: Admin Logging And Regression Coverage

## Objective

Add source-level admin orchestration and test coverage so failed sources are observable, incomplete sources are not falsely marked valid, and the collection model can be rerun safely.

## must_haves

- [ ] Admin routes can start a discovery run without auto-merging
- [ ] Source mismatch/failure details are visible in admin logs/status
- [ ] Tests protect source manifest extraction, incomplete-run persistence, and strict completion validation
- [ ] Runtime History search behavior remains untouched

## Tasks

<task id="01" title="Add admin-run and status surfaces for catalog discovery without auto-merge">
<read_first>
- backend/main.py
- scrape_catalog_discovery.py
- data/catalog_discovery_state.json
- .planning/phases/36-supplemental-catalog-discovery/36-CONTEXT.md
</read_first>

<action>
Extend `backend/main.py` with a dedicated catalog-discovery run path that exposes source-level status and failure logs.

1. Add a new `scraper_status` bucket named `catalog_discovery`.
2. Add a matching `_run_locks` entry for `catalog_discovery`.
3. Add `POST /api/admin/run/catalog-discovery` that:
   - starts `scrape_catalog_discovery.py` in a background thread
   - writes stdout into `log_buffer`
   - updates `scraper_status["catalog_discovery"]`
4. Add `GET /api/admin/run/catalog-discovery/status` that returns:
   - current `scraper_status["catalog_discovery"]`
   - plus the current parsed contents of `data/catalog_discovery_state.json` when present
5. The route must not:
   - call `scrape_merge.py`
   - write to `category_db.json`
   - write to SQLite
   - change `/api/history/products`
6. Failure visibility requirement:
   - if a source is incomplete, the returned status/state must expose `expected_count`, `collected_count`, `complete`, and `last_error`
</action>

<acceptance_criteria>
- `backend/main.py` contains `catalog_discovery` in `scraper_status`
- `backend/main.py` contains `catalog_discovery` in `_run_locks`
- `backend/main.py` contains `@app.post("/api/admin/run/catalog-discovery")`
- `backend/main.py` contains `@app.get("/api/admin/run/catalog-discovery/status")`
- the run route does not call `scrape_merge.py`
- the status route returns source-level fields including `expected_count`, `collected_count`, `complete`, and `last_error`
</acceptance_criteria>
</task>

<task id="02" title="Add regression tests for source-state validity, partial progress, and admin status behavior">
<read_first>
- backend/test_categories.py
- backend/test_history_search.py
- backend/main.py
- scrape_catalog_discovery.py
- .planning/phases/36-supplemental-catalog-discovery/36-VALIDATION.md
</read_first>

<action>
Create `backend/test_catalog_discovery.py` and cover the core Phase 36 contract.

1. Add manifest tests:
   - catalog-root source extraction writes `data/catalog_sources.json`
   - source entries have `name`, `slug`, and `url`
2. Add source-state tests:
   - a source with `collected_count < expected_count` remains `complete = false`
   - a source is marked `complete = true` only after a fresh run with `collected_count == expected_count`
3. Add incremental-progress tests:
   - existing source items survive an incomplete rerun
   - incomplete rerun updates mismatch/error fields instead of wiping progress
4. Add identity tests:
   - preferred numeric `product_id` extraction succeeds for normal product URLs
   - missing/non-numeric identity input is rejected or flagged
5. Add admin-route tests:
   - `POST /api/admin/run/catalog-discovery` starts successfully
   - `GET /api/admin/run/catalog-discovery/status` returns source-state data
6. Keep `backend/test_history_search.py` unchanged and do not add runtime discovery coupling there.
</action>

<acceptance_criteria>
- `backend/test_catalog_discovery.py` exists
- `backend/test_catalog_discovery.py` contains a test where `collected_count < expected_count` keeps `complete = false`
- `backend/test_catalog_discovery.py` contains a test that preserves existing source items across an incomplete rerun
- `backend/test_catalog_discovery.py` contains a test for numeric `product_id` extraction
- `backend/test_catalog_discovery.py` contains a test for `/api/admin/run/catalog-discovery/status`
- `pytest backend/test_catalog_discovery.py backend/test_categories.py backend/test_history_search.py -q` exits 0
</acceptance_criteria>
</task>
