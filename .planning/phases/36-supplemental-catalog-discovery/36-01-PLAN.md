# Plan 01: Source Inventory And Per-Source Collection

## Objective

Implement the core Phase 36 collection model: one manifest of catalog sources, one temp file per source, and one source-state file that records expected count, collected count, completion, and failure reason.

## must_haves

- [ ] Catalog-root sources are discovered from the live VkusVill catalog page
- [ ] Each source is scraped into its own temp file
- [ ] Each source stores progress separately from validity
- [ ] Source validity requires a fresh run that matches the current live count
- [ ] Partial progress survives incomplete runs

## Tasks

<action>
Create the source inventory and state files that Phase 36 will use as its collection contract.

1. Add `scrape_catalog_discovery.py` as the Phase 36 entry script.
2. In that script, define these output paths:
   - `data/catalog_sources.json`
   - `data/catalog_discovery_state.json`
   - `data/catalog_discovery_sources/`
3. Implement catalog-root source discovery from `https://vkusvill.ru/goods/`:
   - collect every visible catalog tile/source link
   - persist each source with:
     - `name`
     - `slug`
     - `url`
     - `source_type`
4. Write `data/catalog_sources.json` with:
   - `updated_at`
   - `source_count`
   - `sources`
5. Define `data/catalog_discovery_state.json` as the source-status file with one entry per source containing:
   - `source_name`
   - `source_slug`
   - `source_url`
   - `expected_count`
   - `collected_count`
   - `complete`
   - `last_run_at`
   - `last_verified_at`
   - `last_error`
   - `last_failed_page`
6. Keep `complete = false` until a fresh source scrape confirms the current live count for that source.
</action>

<task id="02" title="Implement per-source collection with incremental persistence and strict completion validation">
<read_first>
- scrape_categories.py
- .planning/phases/36-supplemental-catalog-discovery/36-CONTEXT.md
- .planning/phases/36-supplemental-catalog-discovery/36-RESEARCH.md
- data/catalog_sources.json
- data/catalog_discovery_state.json
</read_first>

<action>
Implement source-by-source collection in `scrape_catalog_discovery.py`.

1. For each source from `data/catalog_sources.json`:
   - fetch the source page
   - extract the visible source count from the page header text like `1418 товаров`
   - paginate through that source using the same `?PAGEN_1=` pattern already used in `scrape_categories.py`
2. Store collected products for each source in its own file:
   - `data/catalog_discovery_sources/{source_slug}.json`
3. Each source file must contain:
   - `source_name`
   - `source_slug`
   - `source_url`
   - `expected_count`
   - `products`
4. Preserve already scraped items between incomplete runs:
   - load the existing source file if present
   - merge new source items into it
   - do not clear previously collected items just because the run is incomplete
5. Validate the preferred identity key during collection:
   - attempt to use numeric VkusVill `product_id`
   - if a candidate cannot produce the preferred identity key, record that as a source error condition
6. Update `data/catalog_discovery_state.json` after each source run:
   - set `expected_count`
   - set `collected_count` to the unique items collected for that source
   - set `complete = true` only if the current run confirms `collected_count == expected_count`
   - otherwise keep `complete = false` and write `last_error` / `last_failed_page`
7. Do not dedupe across different source files in this task. Cross-source dedupe belongs to a later merge phase.
</action>

<acceptance_criteria>
- `scrape_catalog_discovery.py` reads `data/catalog_sources.json`
- `scrape_catalog_discovery.py` writes files under `data/catalog_discovery_sources`
- each source file contains `expected_count` and `products`
- `scrape_catalog_discovery.py` preserves existing source items on incomplete runs
- `data/catalog_discovery_state.json` sets `complete = true` only when `collected_count == expected_count`
- incomplete sources retain `complete = false` and a non-empty `last_error` or mismatch reason
- no code in this task merges multiple source files into one global file
</acceptance_criteria>
</task>
