# Plan 56-02: xray-core Bootstrap + Subprocess Bridge

## Goal

Download, verify, and manage the lifecycle of an `xray-core` process that exposes a local SOCKS5 listener on `127.0.0.1:10808` and tunnels outbound over a VLESS+Reality pool. No integration with `proxy_manager.py` yet — this plan is a self-contained Python wrapper that `56-03` will call.

## Scope

**In scope:**
- `vless/installer.py` — downloads xray-core release archive for the current OS, verifies SHA256, extracts to `bin/xray/`
- `vless/xray.py` — `XrayProcess` class wrapping `subprocess.Popen`, with start / stop / health-check / graceful restart
- `scripts/bootstrap_xray.py` — CLI entrypoint to install xray on first run
- `bin/` added to `.gitignore`
- Integration test that starts xray with a real config generated from 56-01 and proves outbound RU egress
- Unit tests with mocked subprocess for error paths

**Out of scope (later plans):**
- The vless_manager wrapper that owns a long-lived XrayProcess — 56-03
- systemd unit for EC2 — 56-05
- Scheduler integration — 56-03

## Prerequisites

- 56-01 is complete and merged. `vless.config_gen.build_xray_config(nodes)` returns a valid config dict.
- `httpx` and `httpx[socks]` installed (already in `requirements.txt`).

## xray-core Facts (for the executing AI)

- Repository: https://github.com/XTLS/Xray-core
- Release URL pattern: `https://github.com/XTLS/Xray-core/releases/download/v<VERSION>/Xray-<OS>-64.zip`
  - Windows: `Xray-windows-64.zip`
  - Linux: `Xray-linux-64.zip`
  - macOS: `Xray-macos-64.zip` (dev only — prod is Linux)
- Archive contents (top-level): `xray` or `xray.exe`, `geoip.dat`, `geosite.dat`, `README.md`
- Invocation: `./xray -config /abs/path/to/config.json`
- Runtime behavior: runs in foreground, logs to stdout/stderr
- Signals: SIGTERM for graceful shutdown; SIGINT also works
- Health: once started, the configured inbound SOCKS5 port begins accepting connections within ~1 second
- Version pinning: use a named constant `XRAY_VERSION = "24.11.30"` (check https://github.com/XTLS/Xray-core/releases/latest for current stable at time of execution; pin to one known-good version, do NOT auto-latest)

## Directory Layout

```
bin/                          # gitignored
  xray/
    v24.11.30/                # per-version isolation, allows upgrade w/o corruption
      xray(.exe)
      geoip.dat
      geosite.dat
    current -> v24.11.30      # symlink on Unix; copy on Windows
    configs/
      active.json             # written by vless_manager from 56-03
    logs/
      xray.log                # rotated by Python wrapper
```

## Tasks

### 1. Installer (`vless/installer.py`)

Public API:

```python
from pathlib import Path
from typing import Literal

XRAY_VERSION = "24.11.30"  # pin to a known-good release
XRAY_SHA256 = {
    "windows-64": "...",  # fill in from GitHub release checksums
    "linux-64":   "...",
    "macos-64":   "...",
}
BIN_ROOT = Path(__file__).resolve().parent.parent / "bin" / "xray"

def detect_platform() -> Literal["windows-64", "linux-64", "macos-64"]:
    """Return the xray release slug for the current OS. Raises on unsupported."""

def is_installed(version: str = XRAY_VERSION) -> bool:
    """True if xray binary exists at expected path and passes `xray -version`."""

def install(version: str = XRAY_VERSION, *, force: bool = False) -> Path:
    """
    Download, verify SHA256, extract to bin/xray/v<version>/, update `current` link.
    Returns the path to the xray executable.
    Raises InstallError on download failure, checksum mismatch, or extraction error.
    """

def binary_path(version: str = XRAY_VERSION) -> Path:
    """Return path to xray binary for given version (no check that it exists)."""
```

**Behavior requirements:**
- Uses `urllib.request` (stdlib) to avoid a hard httpx dependency during bootstrap
- Streams the download to disk with progress printed to stderr every ~1MB
- Verifies SHA256 BEFORE extracting — corrupt archives must not reach `bin/`
- Extracts using `zipfile` (Windows/macOS) or `zipfile` + chmod +x (Linux, since zip doesn't preserve mode — explicitly `os.chmod(binary_path, 0o755)`)
- On Linux/macOS, creates `bin/xray/current` as a symlink to `v<version>/`
- On Windows, `current` is a directory junction (via `os.symlink(target_is_directory=True)`) or a copy as fallback
- Idempotent: `install(force=False)` is a no-op if `is_installed()` returns True
- Graceful on download failure: leaves partial files only in a tempdir, never in `bin/`

**Error class:** `InstallError(Exception)` with `cause` chained from the underlying exception.

### 2. Process Wrapper (`vless/xray.py`)

Public API:

```python
from pathlib import Path
from typing import Any
import subprocess
import threading
import time

class XrayProcess:
    """Manages a single xray-core subprocess with its config.

    Thread-safety: methods are thread-safe. Intended to be owned by one
    long-lived VlessProxyManager instance (one xray per Python process).
    """
    def __init__(
        self,
        *,
        binary: Path | None = None,        # default: vless.installer.binary_path()
        config_path: Path,                 # where xray reads its config.json
        log_path: Path | None = None,      # default: bin/xray/logs/xray.log
        restart_limit: int = 3,            # auto-restart attempts per 5min window
        health_check_timeout: float = 5.0, # seconds to wait for SOCKS5 to accept connections
    ): ...

    def start(self) -> None:
        """Start xray and block until the inbound SOCKS5 port is accepting.
        Raises XrayStartupError if xray dies or the port never opens.
        """

    def stop(self, *, timeout: float = 5.0) -> None:
        """Send SIGTERM and wait. If it does not exit in `timeout` seconds, SIGKILL.
        No-op if not running.
        """

    def restart(self, *, new_config: dict | None = None) -> None:
        """Stop, optionally overwrite config, start. Used after pool refresh."""

    def write_config(self, config: dict) -> None:
        """Serialize `config` to `self.config_path` atomically (write-tempfile-rename)."""

    def is_running(self) -> bool:
        """True if the subprocess is alive."""

    def health_check(self) -> bool:
        """Attempt a TCP connect to 127.0.0.1:<inbound-port> within 1s. Returns True if accepting."""

    def verify_egress(self, *, expected_country: str = "RU", timeout: float = 10.0) -> tuple[bool, str]:
        """
        Fetch https://ipinfo.io/json through the local SOCKS5 listener.
        Returns (ok, detected_country_or_error). Used for integration testing.
        """

    @property
    def inbound_port(self) -> int:
        """Extract the socks inbound port from self.config_path (default 10808)."""

class XrayStartupError(RuntimeError): ...
class XrayCrashedError(RuntimeError): ...
```

**Behavior requirements:**
- `start()` spawns `subprocess.Popen([str(binary), "-config", str(config_path)])` with `stdout`/`stderr` redirected to `self.log_path`
- Uses `creationflags=subprocess.CREATE_NEW_PROCESS_GROUP` on Windows so Ctrl-C in parent doesn't nuke xray
- Non-Windows: uses `preexec_fn=os.setsid` so SIGKILL can target the process group
- Spins up a background thread that `.wait()`s on the subprocess — when xray exits unexpectedly, logs the exit code and (if we have spare retries) calls `start()` again after a small backoff
- Restart limit: `restart_limit` attempts in a sliding 5-minute window. After exhaustion, xray stays dead and `is_running()` returns False. The owner (vless_manager) is responsible for noticing and taking action (refresh config, alert, etc.).
- `write_config` uses atomic replace so xray never reads a half-written file
- `health_check` is fast (sub-second). `verify_egress` is slow — only used in tests and diagnostic paths.

### 3. Log Rotation

Simple size-based rotation inside the wrapper:
- When `xray.log` exceeds 5 MB, the wrapper renames it to `xray.log.1` (overwriting any previous `.1`) and starts a new file
- Rotation check runs on every `start()` and every minute via the background watcher thread
- No external dependency (no `logging.handlers.RotatingFileHandler` around subprocess stdout — do it manually)

### 4. CLI (`scripts/bootstrap_xray.py`)

Invocation: `python scripts/bootstrap_xray.py [--force] [--version X.Y.Z]`

Behavior:
1. Detects platform, prints version to install
2. Calls `vless.installer.install(version=..., force=...)`
3. Verifies `xray -version` runs and prints matching version
4. If `--smoke-test`, also:
   - Fetches live VLESS list via `vless.sources.fetch_igareck_list()`
   - Parses, geo-filters, builds config
   - Writes config to `bin/xray/configs/smoke.json`
   - Starts an `XrayProcess`, runs `verify_egress(expected_country="RU")`
   - Stops xray, prints PASS/FAIL summary, exits with 0/1
5. Exit 0 on success, non-zero on failure

## Risks & Mitigations

- **GitHub releases rate-limited for unauthenticated downloads.** → Bootstrap downloads once per install; the binary is cached in `bin/`. Not an issue in practice.
- **Windows symlinks need admin privileges.** → Use directory junctions (which don't need admin) or a directory copy fallback. Detection and fallback are built into `installer.py`.
- **xray binary is flagged by Windows Defender on some machines.** → Document in README that user may need to whitelist `bin/xray/`. Not fixable from Python side.
- **xray version bump breaks config format.** → Version is pinned. New version requires explicit `XRAY_VERSION` change, which is a deliberate code change.
- **Zombie xray on EC2 after deploy.** → `XrayProcess.stop()` uses process group kill (`os.killpg`) so all xray children die. Additionally 56-05's systemd unit sets `KillMode=process` and `Restart=always`.
- **Port 10808 already in use.** → `health_check()` detects this (TCP connect succeeds but behind it is some other process). `start()` catches and raises `XrayStartupError` with a clear "port occupied" message.
