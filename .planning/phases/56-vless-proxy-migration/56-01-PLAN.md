# Plan 56-01: VLESS URL Parser + xray Outbound Config Generator

## Goal

Produce two pure-Python modules — a VLESS URL parser and an xray-core outbound config generator — that together can take the raw VLESS URL list from a public source and emit a valid xray-core JSON config with all ~55 RU-filtered nodes as load-balanced outbounds. No xray process is started in this plan; no integration with `proxy_manager.py` yet.

This is the foundation plan for Phase 56: it converts unstructured external data into a deterministic, testable, reviewable format that every later plan depends on.

## Scope

**In scope:**
- New `vless/` package: `vless/__init__.py`, `vless/parser.py`, `vless/config_gen.py`, `vless/sources.py`
- Unit tests: `tests/test_vless_parser.py`, `tests/test_vless_config_gen.py`
- A fixture file with ~10 representative VLESS URLs covering the param variations we see in the wild
- A CLI self-test (`python -m vless.sources`) that fetches the live igareck list, parses it, geo-filters using existing `scripts/geo_providers.py`, and prints a summary

**Out of scope (later plans):**
- Actually running xray-core — 56-02
- `vless_manager.ProxyManager` — 56-03
- Archival and shim — 56-04
- Production deploy — 56-05

## Tasks

### 1. VLESS URL parser (`vless/parser.py`)

Parse one VLESS URL into a typed structure. The URL format:

```
vless://<uuid>@<host>:<port>?<query>#<fragment>
```

Query params we care about (all optional except `security=reality`):
- `security` (required to be `reality` for our pool)
- `sni` — TLS SNI host
- `fp` — fingerprint (e.g. `chrome`, `firefox`)
- `pbk` — reality public key
- `sid` — reality short-id (may be empty string)
- `spx` — reality spider-x path (may be absent)
- `flow` — e.g. `xtls-rprx-vision`, may be absent
- `type` — transport (usually `tcp`)
- `encryption` — usually `none`
- `headerType` — usually `none`

Return type (proposed):

```python
@dataclass
class VlessNode:
    uuid: str
    host: str
    port: int
    name: str                        # decoded URL fragment, or host:port if empty
    reality_pbk: str
    reality_sni: str
    reality_sid: str = ""
    reality_spx: str = ""
    reality_fp: str = "chrome"
    flow: str = ""
    transport: str = "tcp"
    encryption: str = "none"
    header_type: str = "none"
    extra: dict = field(default_factory=dict)   # any unknown params preserved
```

Parser contract:
- `parse_vless_url(url: str) -> VlessNode` — raises `VlessParseError` on invalid input
- `parse_vless_list(text: str) -> list[VlessNode]` — tolerant line-by-line parse, skips comments / blank lines, aggregates errors but never crashes on a single bad line
- Both preserve unknown query params in `extra` so we never silently drop information

### 2. xray outbound config generator (`vless/config_gen.py`)

Turn a list of `VlessNode` into a full xray-core config JSON (dict serializable to `config.json`). Config must include:

- One `inbound` of type `socks` listening on `127.0.0.1:10808`, no auth, `udp: true`
- One `outbound` per node (tag = `node-<idx>`), type `vless`, with the Reality settings translated from the parsed node
- A `balancer` named `ru-balancer` referencing all node tags with strategy `"random"` (v1 — leastPing is deferred)
- A `routing` rule that routes all traffic to the balancer
- `log.level = "warning"` (production default; plan 56-02 can override for debugging)

Generator contract:
- `build_xray_config(nodes: list[VlessNode], *, listen_host: str = "127.0.0.1", listen_port: int = 10808) -> dict`
- Raises if `nodes` is empty (xray cannot start with no outbounds)
- Output must pass `json.dumps` (no non-JSON types)

### 3. Source fetcher (`vless/sources.py`)

Fetch and parse the upstream list, geo-filter to RU, return ready-to-use nodes.

- `fetch_igareck_list(*, timeout: float = 20) -> str` — returns raw text
- `filter_ru_nodes(nodes: list[VlessNode], *, geo_resolver=None) -> tuple[list[VlessNode], list[VlessNode]]` — returns `(ru_nodes, rejected_nodes)` using the existing `scripts.geo_providers` consensus resolver. Cache-aware.
- CLI (`python -m vless.sources`) — fetch → parse → geo-filter → print `N total / N RU / N rejected / N parse-errors` and sample first 3 RU nodes. Exit code `0` if we get at least 5 RU nodes, non-zero otherwise.

### 4. Tests

**`tests/test_vless_parser.py`** covers:
- Happy-path canonical URL parses correctly
- Missing optional params (`sid`, `spx`, `flow`) default correctly
- URL-encoded fragment (`#%F0%9F%87%B7%F0%9F%87%BA`) decodes
- Invalid scheme / missing uuid / missing host raises `VlessParseError`
- `parse_vless_list` tolerates blank lines, comment lines (`#`), and one bad line in the middle without crashing; returns the good ones and a structured error list

**`tests/test_vless_config_gen.py`** covers:
- Generated config contains exactly one inbound on 127.0.0.1:10808
- N outbounds for N input nodes, each with the correct `vnext[0].address`, `vnext[0].port`, `vnext[0].users[0].id`
- Balancer references every outbound tag
- Empty node list raises
- `json.dumps(config)` round-trips cleanly

Fixture: `tests/fixtures/vless_sample.txt` with ~10 URLs covering: reality full params, reality missing `sid`, reality missing `flow`, URL-encoded fragment, and 2 intentionally malformed lines for error-tolerance tests. Use a placeholder UUID and public test-only host so the fixture is safe to commit.

## Acceptance Criteria

- [ ] `tests/test_vless_parser.py` passes (pytest) with ≥ 6 test cases covering happy path, defaults, URL-encoded name, malformed input, mixed list tolerance, and unknown-param preservation
- [ ] `tests/test_vless_config_gen.py` passes (pytest) with ≥ 4 test cases covering inbound shape, outbound shape, balancer references, and empty-list rejection
- [ ] `python -m vless.sources` against the live igareck list succeeds and reports ≥ 5 RU nodes (we expect ~55 based on prior research)
- [ ] The generated xray config from the live run is saved to a file and structurally validated by a smoke test (no xray invocation yet — that is 56-02)
- [ ] No changes outside `vless/` and `tests/` — `proxy_manager.py` and all callers remain untouched
- [ ] All new Python follows the existing project style (ruff clean per `ruff.toml`)

## Risks & Mitigations

- **Upstream list format drifts.** → Parser is tolerant line-by-line; one bad URL does not kill the run. We log structured errors so format drift is detectable the day after deploy.
- **Geo resolver hits rate limits on first run.** → `scripts/geo_providers.py` already has 10 providers and persistent cache. First run is slow, subsequent runs are instant for unchanged hosts.
- **Unknown Reality params appear in new VLESS URL revisions.** → `extra` dict on `VlessNode` preserves them, config_gen can be extended later without re-parsing.

## Dependencies

- None at the file level. This plan is fully isolated from runtime code and is executable even if `proxy_manager.py` is unchanged.

~1-2 hours. Small, contained, pure-Python with straightforward tests.
