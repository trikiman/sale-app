# Plan 56-04: Archive Legacy SOCKS5 + Install Shim

## Goal

Move the existing `proxy_manager.py` SOCKS5 implementation into `legacy/proxy-socks5/` (preserved, not deleted) and replace it with a thin shim that re-exports `VlessProxyManager` (from 56-03) under the name `ProxyManager`. After this plan, every caller of `from proxy_manager import ProxyManager` transparently uses VLESS. The entire change must be a **single atomic git commit** so rollback is one `git revert`.

## Scope

**In scope:**
- Create `legacy/proxy-socks5/` directory
- Move `proxy_manager.py` → `legacy/proxy-socks5/proxy_manager.py` (preserve full history via `git mv`)
- Move SOCKS5-specific support files to `legacy/proxy-socks5/`:
  - The SOCKS5-specific probe fallback in `_probe_vkusvill_urllib` (still inside proxy_manager.py, moves with it)
  - Any SOCKS5-specific constants that only the legacy code uses
- Create new `proxy_manager.py` (shim, ~20 LOC)
- Create `legacy/README.md` with rollback procedure
- Create `legacy/proxy-socks5/README.md` with SOCKS5 implementation notes
- Run full pytest suite; fix any test that imports `proxy_manager.ProxyManager` incorrectly (there should be zero — the shim makes them all work)

**Out of scope:**
- Any behavior changes beyond the migration
- Deletion of anything — the legacy dir is permanent in repo until a future milestone explicitly removes it
- Production deployment — 56-05
- New data file migration — `data/working_proxies.json` is abandoned (empty/unused) but not deleted; `data/vless_pool.json` becomes the active pool file

## Prerequisites

- 56-01, 56-02, 56-03 complete and merged
- `vless/manager.py` exports `VlessProxyManager`
- All `VlessProxyManager` tests pass

## Design Constraints

### The shim must be SHORT and OBVIOUS

The new `proxy_manager.py` must read like a signpost, not production logic. Every function that existed before must either:
- Re-export from `vless.manager`, OR
- Be defined here as a trivial pass-through

**Size target:** ≤ 30 lines of executable code. More than that = there's extra logic that belongs in `vless.manager`.

### The rollback must be verifiable

After the commit, running `git revert <commit>` must cleanly:
1. Restore `proxy_manager.py` to the original SOCKS5 implementation
2. Delete `legacy/proxy-socks5/proxy_manager.py`
3. Delete `legacy/README.md` and `legacy/proxy-socks5/README.md`
4. Produce a working codebase where `pytest` still passes (with the SOCKS5 tests back in force)

This is testable: in the branch, commit the change, then `git revert` it on a throwaway branch and run `pytest`. Include a documented rehearsal in 56-05.

### 3. Write New Shim (`proxy_manager.py`)

Contents (adjust if `vless.manager.VlessProxyManager` ended up with a different name):

```python
"""Compatibility shim: proxy_manager → vless_manager (VLESS+Reality migration).

v1.15 replaced the free SOCKS5 pool with a VLESS+Reality pool behind a local
xray-core SOCKS5 bridge. The old implementation lives under
``legacy/proxy-socks5/proxy_manager.py`` (preserved for one-operation rollback).

For the legacy behavior, see ``legacy/README.md`` and run:

    git revert <commit-of-this-shim>

Callers do not need any code changes — every public method of the old
``ProxyManager`` is preserved on ``VlessProxyManager`` via 56-03 contract.
"""
from vless.manager import VlessProxyManager as ProxyManager

__all__ = ["ProxyManager"]
```

That's it. If the executing AI is tempted to add "just one helper" to this file — stop. It belongs in `vless/manager.py`.

This is the v1.0–v1.14 proxy implementation. It fetched free SOCKS5 proxies
from public lists (primarily `proxifly/free-proxy-list`), tested them against

## Why It Was Retired

- Free SOCKS5 pool reliability collapsed — 0% alive on 2026-04-22 across
  269 tested nodes (see `.cache/alive_ru_proxies.json`).
- Add-to-cart and green-scraper failures were traced to pool exhaustion,
  not to cart or scraper logic bugs.
- Replacement: VLESS+Reality pool with ~55 alive RU nodes and daily refresh,
  tunneled through a local xray-core SOCKS5 bridge.

## Public API (preserved by VLESS replacement)

`vless/manager.py` reproduces every public method with compatible
signatures — callers do not need changes.

## Data Files Used (no longer read)

- `data/working_proxies.json` — SOCKS5 pool cache
- `data/proxy_events.jsonl` — event log (schema preserved, still written
  by the VLESS manager)
- `.cache/vkusvill_cooldowns.json` — 4h cooldown cache (schema and file
  path preserved; VLESS manager uses the same cache)

## If You Need This Code Back

Use `git revert` on the archive commit — do not copy-paste back. See
`../README.md` for the exact rollback procedure.

## Risks & Mitigations

- **A test relies on a private SOCKS5 method name.** → Move the test under `tests/legacy/` so it runs against `legacy/proxy-socks5/proxy_manager.py` if needed, or rewrite the test against the public API. Do NOT expose SOCKS5 private methods on `VlessProxyManager`.
- **A caller imports `SOCKS5_CONNECT_TIMEOUT` or another legacy constant.** → Grep for each constant name. If found in production, either export it from the shim (preserved constant re-export) or update the caller to use the new VLESS equivalent.
- **git mv produces a diff that git interprets as delete+add instead of rename.** → Commit the move BY ITSELF first (no other changes). Verify with `git log --follow`. Then amend in the shim in a follow-up commit. OR do them together and verify rename detection worked post-commit. Either approach preserves history if done cleanly.
- **Revert fails because of conflicting changes in the same commit.** → Keep the commit minimal — only the move, the shim, and the two READMEs. No other changes.
