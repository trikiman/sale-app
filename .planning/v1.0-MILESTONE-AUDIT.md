## Critical Gap: SCRP-07

### Problem

The green scraper on EC2 captures **8 items** and reports `greenLiveCount: 8`, giving a false 100% accuracy ratio.
The actual VkusVill site shows only **2 green-tagged items** (confirmed via search: "Зелёный ценник — 2 результата поиска").

### Root Causes

1. **`greenLiveCount` detection is broken** — the scraper counts wrong DOM elements on the VkusVill page (badges, cards, or other elements that aren't exclusively green-tagged items)
2. **No eviction mechanism** — items that were once green-tagged but lost that status persist in `green_products.json` across runs
3. **The "suspicious result protection"** that rejects empty/single-item results may be PRESERVING stale data when the true count drops to 2
