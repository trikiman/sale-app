## Executive Summary

v1.19 delivers the full robust-over-fast hardening contract the user demanded after the post-v1.18 production drift exposure (pool 25→13, 162 useless breaker re-trips, 30/30 detail-proxy timeouts behind a 200-OK Vercel cache, plus the PR #25 8-minute-revert). All three phases shipped with rehearsed rollback, EC2 smoke verification, and live evidence. The final mainline state delivers:

- A **truthful unauthenticated deep-health endpoint** at `https://vkusvillsale.vercel.app/api/health/deep` returning HTTP 200 healthy / 503 degraded|unhealthy with `reasons[]`, suitable for any external uptime monitor (UptimeRobot, BetterStack, Datadog).
- A **graduated 3-state circuit breaker** with exponential backoff (120 s → 30 min cap) persisted across scheduler restarts, replacing the cycle-counter that produced 162 re-trips in 5.4 h.
- A **pre-flight VLESS probe** with 12 s timeout (vs PR #25's failed 5 s) and rotation cap of 2 (vs unbounded), eliminating the 30-45 s wasted-Chrome-launch pattern.

- **Pool drift visibility** via `VlessProxyManager.pool_snapshot()` and auto-enriched `proxy_events.jsonl` (every event carries `pool_size`, `quarantined_count`, `active_outbounds_count`).

## Tech Debt (non-blocking)

Two items captured during pre-audit tightening; neither is a deliverable miss:

1. **OBS-02 wording vs runtime file naming** — REQUIREMENTS.md OBS-02 mentions "merged products.json" but the actual file the scheduler maintains and the API serves is `data/proposals.json`. The constant `_MERGED_PRODUCTS_PATH` aliases `PROPOSALS_PATH` with an explanatory comment. Spec wording could be amended in a future cleanup; runtime is correct.
2. **Smoke 60-G window relaxation** — The 3-state breaker is silent in steady-state; the original 10-min journal window flagged false negatives on healthy nodes. Widened to 60 min — keeps the integration-live signal without flagging healthy.
3. **Phase 61 README ↔ REQUIREMENTS.md drift** — README captured a simpler 4-criteria healthy contract; OBS-02 in REQUIREMENTS.md was the authoritative 5-criteria contract. Implementation, tests, smoke, and verification doc all align with REQUIREMENTS.md (commit `42158ea`). README itself is left as a snapshot of phase intent at planning time, with the verification doc reflecting final delivery.

None of these block milestone close-out.
