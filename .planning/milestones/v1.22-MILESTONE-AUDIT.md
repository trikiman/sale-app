## Findings

### Strengths

- **Cohesive scope, tight execution.** 4 small phases (~20-100 LOC each in product code) mirror the v1.21 discipline. No scope creep despite two of the phases being in miniapp and one being tooling.
- **Cross-milestone integration respected.** v1.21's `xray_drift` surface naturally folds into v1.22's admin badge scope. v1.5's `product_catalog` table is reused in v1.22 Phase 70's search enrichment. v1.16's `bugReports` payload is consumed by v1.22 Phase 72 without a backend round-trip.
- **Live verification discipline carried over from v1.21.** Phases 70/71/72 all have live MCP runbooks + smoke script live-verified green on EC2. Lesson learned from v1.21's initial "code-complete but never deployed" phase is holding.
- **Test suite growth tracked:** 355 → 360 (+5 from Phase 70). Phases 71/72/73 didn't add unit tests by design (frontend copy, HTML badge, Kiro-side CLI) — tech debt lines captured for v1.23 if needed.

### Gaps / Tech Debt

No true requirement gaps. All items captured are:
- Phases 70/71/72 NEEDS_OPERATOR UI screenshots (visual proof of live rendering — deferred to operator).
- Phase 72 admin endpoint E2E with ADMIN_TOKEN env var (smoke 72-D skips gracefully when not set).
- Phase 73 Kiro-side files sit outside the repo; rollback uses `.bak-pre73` siblings. Documented.
- Multi-select fold-into-milestone (TOOL-01 partial) — deferred to v1.23+ when Kiro tooling supports it.

### Risks for production

- **Backend SSE graceful-shutdown loop.** Observed during Phase 72 live verification — `/api/stream` connections hold the socket through systemd's 90s stop-sigterm timeout, chaining with a second github-webhook restart caused a 4-minute backend gap. Pre-existing behavior, not v1.22-specific, but worth a future reliability phase.
- **Vercel cache propagation.** After pushing Phase 70/71 miniapp changes, users see the old bundle for 2-3 min. Acceptable at family-scale but flagged.
- **Admin-only changes in Phase 72.** `/admin` is EC2-served (not Vercel-proxied), so miniapp cache lag doesn't affect this phase — but operators need to hard-refresh the admin tab to see the new badges.
- **Phase 73 Kiro-side edits.** CLI changes apply immediately to subsequent `/gsd-*` skill runs, but SKILL.md + workflow changes may need a Kiro reload to take effect. Documented in 73-VERIFICATION.md.
