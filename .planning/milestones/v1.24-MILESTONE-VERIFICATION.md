## Phase 77 — Pool Self-Heal Hardening

### Mechanism soundness

Three layers:
1. **Persistent quarantine** — 231 RU-filtered dead nodes on 2026-05-13 no longer re-enter the candidate set for 20 min. Correct lever — original outage's tail was dominated by repeated probing of known-dead nodes.
2. **Refresh throttle (60s)** — prevents collapsed pool from triggering thrash-refresh. Correct pattern.
3. **`MIN_HEALTHY=10` + rate-of-decline** — detects collapse earlier than `size==0` gate.

### Gaps / concerns

- **HIGH → RESOLVED in code**: Quarantine layer placement vs. RU filter. Confirmed: `refresh_proxy_list` in `vless/manager.py` reads `quarantine.get_quarantined_hosts()` AFTER the RU filter produces `ru_nodes`. The candidate-filter loop checks `node_key in quarantined_host_ports` as part of the per-node filter with `cooling` and `dead_hosts`. Quarantine is the final gate before `_probe_candidates_in_parallel`. ✓
- **HIGH → PARTIALLY RESOLVED**: Rate-of-decline test coverage does not match observed 20→0 collapse shape. Unit test exercises gradual 20→17, not 20→0 in one cycle. But the early-warning logic is trivially triggered by 20→0 (below threshold immediately). The *missing* test is the integration test "20→0 → refresh → quarantine skips 231 → recovery ≤10 min." **Carried as v1.25 enhancement**, not gating archive since the design mechanism is independently verifiable.
- **HIGH → RESOLVED in code**: Scheduler graceful-degrade race. `_is_pool_dead` reads `vless_pool.json` directly. Confirmed `pool_state.save(self._pool, self._pool_path)` in `manager.py` uses `tmp + os.replace` pattern (via `pool_state` module). Atomic. Scheduler won't see transient empty pool. ✓
- **MEDIUM — Deferred**: 60s refresh throttle during genuine recovery. Worst case: mid-recovery network blip → 60s delay before next retry. Low-probability, not blocking. Flagged for v1.25 observation.
- **MEDIUM — Deferred**: No admin endpoint to force-clear `pool_quarantine.json`. 20-min TTL + `quarantine.clear_all()` utility exists; can be invoked via `python3 -c "from vless import quarantine; quarantine.clear_all()"` on EC2 in emergency. Escape hatch good enough; adding HTTP endpoint is polish.
- **LOW — Clock consistency**: `_monotonic` static is used for throttle + size-history. `quarantine.py` uses `time.time()` for TTL (wall clock — correct for TTL semantics, monotonic wouldn't survive process restart). Confirmed mix is intentional and correct.

### Verdict: PASS with v1.25 carry-forward
