## Milestone Goal (as delivered)

Kept the VkusVill sale app continuously healthy from the user's perspective (Vercel frontend + Telegram MiniApp) 24/7 by hardening the EC2 data pipeline against the failure modes that regressed post-v1.18 — with regression tests and per-phase EC2 smoke verification that would have caught PR #25's revert in pre-merge.

Driving evidence at milestone start (2026-05-03):
- Pool drift 25 → 13 over 8 days post-v1.18
- 162 consecutive scraper-cycle failures; fixed cooldown re-tripped every 2 min for 5.4 h with no self-recovery
- 30/30 detail-proxy timeouts while Vercel `/api/products` returned HTTP 200 cached data
- PR #25 (5 s pre-flight) reverted after 8 min because empirical healthy p95 is 7-9 s
