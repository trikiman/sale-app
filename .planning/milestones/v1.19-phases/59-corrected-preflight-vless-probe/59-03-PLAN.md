## Implementation Steps

### Step 1 — Create `scripts/verify_v1.19.sh`

```bash
#!/usr/bin/env bash
# v1.19 Reliability Smoke Test
# -----------------------------
# Runs from local terminal. SSHes to the EC2 scraper host. Reports pass/fail
# per criterion. Exits 0 on all-pass, 1 on any-fail. Idempotent.
#
# Usage:
#   ./scripts/verify_v1.19.sh           # runs all phases
#   ./scripts/verify_v1.19.sh 59        # runs only Phase 59 checks
#
# Grows with each phase. Each phase block is gated so partial runs are safe.
#
# Requires: SSH access to `scraper-ec2` host (configure in ~/.ssh/config).
# The Vercel-facing check requires outbound HTTPS from the local machine.

set -uo pipefail

PHASE="${1:-all}"
EC2_HOST="scraper-ec2"
VERCEL_BASE="https://vkusvillsale.vercel.app"
FAILED=0

_pass() { printf '  \033[32m✓\033[0m %s\n' "$1"; }
_fail() { printf '  \033[31m✗\033[0m %s\n' "$1"; FAILED=1; }

_check_ec2_ssh() {
    ssh -o BatchMode=yes -o ConnectTimeout=5 "$EC2_HOST" "echo ok" >/dev/null 2>&1
}

_banner() {
    echo ""
    echo "=== $1 ==="
}

_banner "v1.19 Smoke Test (phase: $PHASE)"

if ! _check_ec2_ssh; then
    _fail "Cannot SSH to $EC2_HOST — configure ~/.ssh/config first"
    exit 1
fi
_pass "SSH to $EC2_HOST reachable"

# ---------------------------------------------------------------------------
# Phase 59: Corrected Pre-flight VLESS Probe
# ---------------------------------------------------------------------------
if [[ "$PHASE" == "59" || "$PHASE" == "all" ]]; then
    _banner "Phase 59 — Pre-flight VLESS Probe"

    # 59-A: vless/preflight.py deployed
    if ssh "$EC2_HOST" "test -f /home/ubuntu/saleapp/vless/preflight.py"; then
        _pass "59-A: vless/preflight.py exists on EC2"
    else
        _fail "59-A: vless/preflight.py MISSING on EC2 — deploy incomplete"
    fi

    # 59-B: module imports cleanly (no ImportError)
    if ssh "$EC2_HOST" "cd /home/ubuntu/saleapp && python3 -c 'from vless.preflight import probe_bridge_alive, ProbeResult' 2>/dev/null"; then
        _pass "59-B: vless.preflight imports without error"
    else
        _fail "59-B: vless.preflight FAILS to import — check httpx kwarg compat"
    fi

    # 59-C: timeout floor constant is still >= 12.0 (regression guard)
    FLOOR=$(ssh "$EC2_HOST" "cd /home/ubuntu/saleapp && python3 -c 'from vless import preflight; print(preflight._PROBE_TIMEOUT_S_FLOOR)'" 2>/dev/null)
    if awk "BEGIN {exit !($FLOOR >= 12.0)}"; then
        _pass "59-C: _PROBE_TIMEOUT_S_FLOOR = $FLOOR (>= 12.0)"
    else
        _fail "59-C: _PROBE_TIMEOUT_S_FLOOR = $FLOOR (< 12.0) — PR #25-style regression"
    fi

    # 59-D: scheduler log shows pre-flight probe invoked in recent history
    if ssh "$EC2_HOST" "sudo journalctl -u saleapp-scheduler -n 500 --no-pager | grep -q 'Pre-flight probe'"; then
        _pass "59-D: scheduler invokes pre-flight probe (log evidence)"
    else
        _fail "59-D: no 'Pre-flight probe' log in last 500 lines — integration not active"
    fi

    # 59-E: unit test suite still green
    if ssh "$EC2_HOST" "cd /home/ubuntu/saleapp && python3 -m pytest tests/test_preflight_timeout_regression.py tests/test_preflight_probe_contract.py tests/test_preflight_rotation_cap.py -q 2>&1 | tail -5 | grep -q 'passed'"; then
        _pass "59-E: preflight pytest suite green on EC2"
    else
        _fail "59-E: preflight pytest suite FAILED on EC2"
    fi

    # 59-F: xray is running (systemd)
    if ssh "$EC2_HOST" "systemctl is-active saleapp-xray" | grep -q '^active$'; then
        _pass "59-F: saleapp-xray is active"
    else
        _fail "59-F: saleapp-xray is NOT active"
    fi

    # 59-G: scheduler is running
    if ssh "$EC2_HOST" "systemctl is-active saleapp-scheduler" | grep -q '^active$'; then
        _pass "59-G: saleapp-scheduler is active"
    else
        _fail "59-G: saleapp-scheduler is NOT active"
    fi

    # 59-H: Vercel miniapp /api/products returns HTTP 200 (smoke the public edge)
    CODE=$(curl -s -o /dev/null -w "%{http_code}" --max-time 20 "$VERCEL_BASE/api/products")
    if [[ "$CODE" == "200" ]]; then
        _pass "59-H: Vercel /api/products returns HTTP 200 (edge reachable)"
    else
        _fail "59-H: Vercel /api/products returns HTTP $CODE (edge down)"
    fi

    # 59-I: Vercel miniapp cart-add still works (no v1.18 regression).
    # Uses an anonymous-safe endpoint check: HEAD-probe the backend-proxied
    # /api/cart/* path exists. Full cart-add requires a bound session and is
    # covered by the manual check in 59-VERIFICATION.md.
    CODE=$(curl -s -o /dev/null -w "%{http_code}" --max-time 15 -X OPTIONS "$VERCEL_BASE/api/cart/add")
    if [[ "$CODE" == "200" || "$CODE" == "204" || "$CODE" == "405" ]]; then
        _pass "59-I: Vercel /api/cart/add endpoint reachable (CORS preflight $CODE)"
    else
        _fail "59-I: Vercel /api/cart/add unreachable (got $CODE) — investigate"
    fi
fi

# ---------------------------------------------------------------------------
# Future phases 60, 61 append their blocks here:
#
# if [[ "$PHASE" == "60" || "$PHASE" == "all" ]]; then
#     _banner "Phase 60 — Observatory probeURL + Graduated Circuit Breaker"
#     ...
# fi
#
# if [[ "$PHASE" == "61" || "$PHASE" == "all" ]]; then
#     _banner "Phase 61 — Deep Health Endpoint + Pool Snapshot"
#     ...
# fi
# ---------------------------------------------------------------------------

_banner "Summary"
if [[ $FAILED -eq 0 ]]; then
    _pass "All checks passed for phase '$PHASE'"
    exit 0
else
    _fail "One or more checks failed — review output above"
    exit 1
fi
```

Make it executable:


### Step 2 — Rehearse rollback on a throwaway branch (OPS-08)

**Critical:** Do this BEFORE merging 59-01 + 59-02 to `main`.

```bash
# On local machine:
git checkout -b phase-59-rehearsal-rollback
# 59-01 + 59-02 commits already exist on the branch from prior plans

# Push the rehearsal branch, deploy to EC2 from the branch
git push origin phase-59-rehearsal-rollback
ssh scraper-ec2 "cd /home/ubuntu/saleapp && git fetch origin && git checkout phase-59-rehearsal-rollback && sudo systemctl restart saleapp-scheduler"

# Observe ~3 cycles (15 min) via journalctl
ssh scraper-ec2 "sudo journalctl -u saleapp-scheduler -f"

# Simulate the need to roll back: revert the integration commit
ssh scraper-ec2 "cd /home/ubuntu/saleapp && git revert HEAD --no-edit && sudo systemctl restart saleapp-scheduler"

# Verify scheduler still runs cleanly after revert (no stuck imports, no
# dangling references to probe_bridge_alive)
ssh scraper-ec2 "sudo journalctl -u saleapp-scheduler -n 50 --no-pager"

# Redeploy the phase-59 commits (undo the revert)
ssh scraper-ec2 "cd /home/ubuntu/saleapp && git revert HEAD --no-edit && sudo systemctl restart saleapp-scheduler"
```

If both the revert AND the re-apply leave the scheduler running cleanly, rollback is rehearsed. Document evidence in `59-VERIFICATION.md` Section "Rollback Rehearsal" with timestamps.

After rehearsal:
```bash
# On local machine, merge the branch to main via standard PR flow

git merge phase-59-rehearsal-rollback --ff-only
git push origin main
git branch -d phase-59-rehearsal-rollback
git push origin --delete phase-59-rehearsal-rollback
```

### Step 4 — Observation window: watch one full cycle

```bash
ssh scraper-ec2 "sudo journalctl -u saleapp-scheduler -f"
```

Watch for:

1. `Pre-flight probe: ok (status=200, 8.3s)` — normal path, probe passes
2. No `Pre-flight probe still failing after 2 rotations` — if seen, either the bridge is genuinely down OR the probe is mis-calibrated; investigate before claiming success
3. Scraper tags (RED/YELLOW/GREEN) still report `OK (exit 0)` — no regression on the scrape itself
4. Merge step still runs (`MERGE: OK (data updated)`)
5. Vercel miniapp still serves fresh data within 15 min

Let it run for at least 2 full cycles (~10 min) before ending the observation.

### Step 5 — Manual Vercel cart-add regression check

The smoke script's 59-I is a CORS preflight proxy for the cart endpoint. The real regression check is a live cart-add through the miniapp:

1. Open `https://vkusvillsale.vercel.app` in a browser (Telegram WebApp or plain browser)
2. Pick any product with a green sale tag
3. Click "Add to cart"
4. Confirm the response is HTTP 200 with `success=true` (check browser DevTools Network tab OR the miniapp's in-UI feedback)
5. Screenshot evidence → commit to `.planning/phases/59-corrected-preflight-vless-probe/59-cart-add-evidence.png`
