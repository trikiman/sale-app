## Step 1 — Extend `scripts/verify_v1.19.sh` with Phase 60 checks

Append a new phase block between the Phase 59 block and the `"future phases"` comment:

- **60-A**: `scheduler_state.json` gitignored? (`grep -q 'scheduler_state.json' .gitignore` — otherwise it'll show up in `git status`)
- **60-B**: xray probeURL references `vkusvill.ru` on EC2 (`python3 -c 'from vless import config_gen; ...'`)
- **60-C**: Live xray config pulled from running daemon includes `vkusvill.ru` in observatory.probeURL (via `xray api statsquery` if exposed; fallback: re-build config offline and compare)
- **60-D**: `_PROBE_TIMEOUT_S_FLOOR` still >= 12 (Phase 59 regression guard still passing after Phase 60 edits)
- **60-E**: Breaker state machine pytests green on EC2
- **60-F**: `scheduler_state.json` exists on EC2 after one cycle + is valid JSON + `state` field is one of `{closed, open, half_open}`
- **60-G**: `journalctl -u saleapp-scheduler --since '10 min ago' | grep -cE 'Circuit breaker CLOSED|OPEN|HALF_OPEN'` > 0 when breaker is active (skip-gate if breaker has been closed the whole window)

Exact snippet:

```bash
if [[ "$PHASE" == "60" || "$PHASE" == "all" ]]; then
    _banner "Phase 60 — Observatory probeURL + Circuit Breaker"

    # 60-A
    if grep -q '^data/scheduler_state.json\|^scheduler_state.json' .gitignore 2>/dev/null; then
        _pass "60-A: data/scheduler_state.json is gitignored"
    else
        _fail "60-A: data/scheduler_state.json NOT in .gitignore — it will pollute git status"
    fi

    # 60-B
    PROBE=$(ssh "$EC2_HOST" "cd /home/ubuntu/saleapp && python3 -c 'from vless.config_gen import build_xray_config; from vless.config_gen import VlessNode; n = VlessNode(host=\"x\",port=443,uuid=\"00000000-0000-0000-0000-000000000000\",flow=\"\",sni=\"x\",fp=\"chrome\",public_key=\"x\",short_id=\"00\"); print(build_xray_config([n])[\"observatory\"][\"probeURL\"])'" 2>/dev/null)
    if [[ "$PROBE" == *"vkusvill.ru"* ]]; then
        _pass "60-B: observatory.probeURL = '$PROBE'"
    else
        _fail "60-B: observatory.probeURL = '$PROBE' (expected vkusvill.ru)"
    fi

    # 60-D
    FLOOR=$(ssh "$EC2_HOST" "cd /home/ubuntu/saleapp && python3 -c 'from vless import preflight; print(preflight._PROBE_TIMEOUT_S_FLOOR)'" 2>/dev/null)
    if awk "BEGIN {exit !($FLOOR >= 12.0)}" 2>/dev/null; then
        _pass "60-D: Phase 59 timeout floor still $FLOOR (>= 12.0)"
    else
        _fail "60-D: Phase 59 floor regressed to $FLOOR — Phase 60 should not touch preflight.py"
    fi

    # 60-E
    if ssh "$EC2_HOST" "cd /home/ubuntu/saleapp && python3 -m pytest tests/test_xray_probe_url_regression.py tests/test_circuit_breaker_state_machine.py -q 2>&1 | tail -3 | grep -q 'passed'"; then
        _pass "60-E: breaker + probeURL pytests green on EC2"
    else
        _fail "60-E: breaker + probeURL pytests FAILED on EC2"
    fi

    # 60-F
    STATE=$(ssh "$EC2_HOST" "cat /home/ubuntu/saleapp/data/scheduler_state.json 2>/dev/null | python3 -c 'import json,sys; d=json.load(sys.stdin); print(d.get(\"state\",\"<missing>\"))'" 2>/dev/null)
    case "$STATE" in
        closed|open|half_open)
            _pass "60-F: data/scheduler_state.json valid (state=$STATE)"
            ;;
        *)
            _fail "60-F: data/scheduler_state.json invalid or missing (state=$STATE)"
            ;;
    esac
fi
```

## Step 5 — Live transition rehearsal

Force all 3 scrapers to fail for 3 cycles (via a script that temporarily blocks vkusvill.ru at the host firewall — or, less invasive, rename the scraper data dir to induce failure) and observe:

```bash
ssh scraper-ec2 "sudo journalctl -u saleapp-scheduler --since '15 minutes ago' -n 200 --no-pager | grep -E 'Circuit breaker|cycle|GREEN|RED|YELLOW'"
```

Expected pattern over ~10 minutes:

```
[T+0]   === Starting Full Scrape Cycle ===
[T+1]   RED: FAILED, YELLOW: FAILED, GREEN: FAILED
[T+1]   (no breaker log yet — 1/3)
[T+5]   === Starting Full Scrape Cycle ===
[T+6]   RED: FAILED, YELLOW: FAILED, GREEN: FAILED  (2/3)
[T+10]  === Starting Full Scrape Cycle ===
[T+11]  Circuit breaker CLOSED -> OPEN (3 all-fail cycles; cooldown=120s)
[T+13]  Circuit breaker OPEN: 120s until half_open (cooldown_s=120)
[T+13]  (sleeps)
[T+13]  === Starting Full Scrape Cycle ===  <- ticks, now half_open
[T+13]  Circuit breaker HALF_OPEN: running GREEN-only probe
[T+14]  GREEN: OK (assume bridge recovered)
[T+14]  Circuit breaker HALF_OPEN -> CLOSED (green probe succeeded)
```

Total recovery ≤ 5 min vs. 5.4 h in v1.18 baseline (success criterion #6).
