# v1.12 Milestone Audit: Add-to-Cart 5s Hard Cap

**Audited:** 2026-04-22
**Status:** `passed_superseded`
**Auditor:** Cascade (retroactive closure)

## Audit Context

v1.12 was marked `✅ shipped` in `.planning/ROADMAP.md` on 2026-04-08 but was never formally audited or archived. This closure is a retroactive audit performed four milestones later (v1.13, v1.14, and the Apr 22 scheduler hotfix all shipped in between).

Two behaviors from v1.12 are relevant to today's code:

1. **Architecturally intact**: the AbortController + time-budget pollCartAttemptStatus pattern from phase 46 is still present in `miniapp/src/App.jsx` (confirmed on 2026-04-22).
2. **Numerically superseded**: the literal 5000 ms / 4000 ms budget constants were relaxed to 8000 ms / 7000 ms in v1.13 (commit `5d16e14` — "separate poll budget (8s own clock) from initial request (8s) — no more shared 5s starvation"). The original v1.12 5-second cap was shipped, tested in practice, and found too tight. The tuned values replaced it.

This audit treats the milestone as **shipped and superseded**, not failed.
