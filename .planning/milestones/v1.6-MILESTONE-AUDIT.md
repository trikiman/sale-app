### SCRP-13: live_count vs scraped_count validation gate ✅

**Status:** SATISFIED

**Evidence:**
- Validation gate at line ~2469-2486, tagged with `# SCRP-13`
- Log output confirms: `⚠️ [GREEN] COUNT MISMATCH: scraped=12, live=164, gap=93%`
- When gap >10%: `⚠️ [GREEN] Existing snapshot has 111 items — preserving it`
- Existing `green_products.json` with 111 items was NOT overwritten by 12-item failed runs
- When gap ≤10%: saves normally with `Count OK: scraped=N, live=N, gap=X%`

**Implementation:**
- `gap_pct = abs(live_count - scraped_count) / live_count * 100`
- If `scraped_count < live_count and gap_pct > 10`: refuse to save, preserve existing
- If existing snapshot has MORE items: `scrape_success = False`
