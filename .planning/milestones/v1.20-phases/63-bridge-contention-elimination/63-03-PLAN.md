### Step 3 - 50-Sample Cart-Add Latency (during active scraper window)

Trigger a scraper cycle on EC2:

```bash
# Force a detail-fetch scrape so the bridge is actively contended
ssh "$EC2_HOST" "cd /home/ubuntu/saleapp && python3 scrape_green.py &"
```

Then from the operator laptop, hit `/api/cart/add` 50 times (~1 req / 2 s) against a known test user + in-stock product. Record each latency from HTTP timing. Python one-liner:

```bash
python3 - <<'PY'
import json, time, statistics, urllib.request
url = "https://vkusvillsale.vercel.app/api/cart/add"
user_id = "<test-user-id>"
product_id = <test-product-id>
body = json.dumps({"user_id": user_id, "product_id": product_id, "price_type": 1, "is_green": 0, "allow_pending": True}).encode()
samples = []
for i in range(50):
    t = time.monotonic()
    req = urllib.request.Request(url, data=body, headers={"Content-Type": "application/json", "X-User-Id": user_id})
    with urllib.request.urlopen(req, timeout=30) as r:
        r.read()
    samples.append(time.monotonic() - t)
    time.sleep(2)
samples.sort()
p50 = samples[24]
p95 = samples[int(0.95 * len(samples))]
print(f"p50={p50*1000:.0f}ms p95={p95*1000:.0f}ms max={max(samples)*1000:.0f}ms")
PY
```

Record: `p50`, `p95`, `max`. Paste into `63-VERIFICATION.md`.

**Gate:** `p95 <= 4.5 s` AND `p95 <= baseline + 500 ms` (baseline = Phase-62 p95 captured on same EC2 host).
