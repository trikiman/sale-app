# 62-01: `keepalive/warmup.py` Module + Scheduler Integration

**Delivers:** PERF-03 (20-min background keep-alive for every linked user), PERF-05 (anti-spam <=1 req/user/15 min + warmup p95 <= 3 s), OPS-09 skeleton (emits `data/warmup_events.jsonl` consumed by `scripts/verify_v1.20.sh` in 62-02).

**Estimated effort:** 120 minutes — one new module (~180 LOC), one 3-line spawn in `scheduler_service.main()`, no other code changes.

**Risk:**
- Low on `keepalive/warmup.py`: new module, no existing callers.
- Medium on `scheduler_service.py`: wrong spawn position or a thread that leaks exceptions could compromise the scheduler loop. Mitigated by running everything in `try/except` and making the outer loop self-contained.

***

## Goal

Stand up the 20-min background keep-alive daemon thread that refreshes every linked user's VkusVill sessid so the cart-add hot path never has to do an inline `_refresh_stale_session`. This is the core engine that 62-02 (backend nudge triggers + smoke checks) and 62-03 (EC2 deploy + live verification) build on.

Honors LOCKED decisions D1 (warm ALL linked users every 20 min — no activity filter), D2 (silent log + JSONL on failure, NO inline stale mark, NO Telegram), D3 (daemon thread inside `scheduler_service.py`, new module `keepalive/warmup.py`), D4 (cart-add cancels in-flight warmup via per-user `CART_ADD_ACTIVE` flag — this plan exports the flag registry; 62-02 wires the cart-add side).

***

## Success Criteria (observable / testable)

1. `python -c "from keepalive.warmup import start_warmup_loop, CART_ADD_ACTIVE, NUDGE_QUEUE"` succeeds without `ImportError`.
2. Starting `scheduler_service.main()` in a subprocess produces a thread named `scheduler-keepalive` visible in `threading.enumerate()` within 2 s of boot and still alive after 65 s (survives the 60-s boot grace).
3. With a mocked `_refresh_stale_session` that sets a known sessid, running one cycle end-to-end over a fixture of 3 linked users writes 3 JSONL lines to `data/warmup_events.jsonl`, each with keys `{timestamp_iso, user_id_hash, trigger, endpoint, success, outcome, latency_ms, sessid_changed}` (8 keys total) — schema verified by `tests/test_keepalive_warmup.py::test_jsonl_schema` (62-02).
4. With `pool_snapshot()` monkeypatched to `{"size": 4, "quarantined_count": 3, ...}`, one cycle writes exactly one `"outcome": "skipped_unhealthy"` line per user and performs zero HTTP calls.
5. Setting `stop_event.set()` causes the thread to return within 5 s regardless of where it is in the iteration loop.
6. Anti-spam: running two cycles 10 min apart (fast-forwarded clock) produces 1 real attempt + 1 `"skipped_recent"` per user, proving the <=1/user/15 min ceiling holds across scheduler triggers.

### Step 2 — Create `keepalive/warmup.py` (~180 LOC)

The module exposes:

- `CART_ADD_ACTIVE: dict[str, float]` — module-level registry. Keys are `user_id_hash`, values are `time.monotonic()` of when the cart-add started. Read/write protected by `_STATE_LOCK = threading.Lock()`. Consumed by 62-02's cart-add edit.
- `NUDGE_QUEUE: queue.Queue[str]` — module-level `queue.Queue(maxsize=256)` of `user_id_hash`. The backend writes into this queue from `/api/link/status` and `/api/cart/items` handlers (62-02 wires the backend side). The daemon drains it via `NUDGE_QUEUE.get(timeout=NUDGE_POLL_INTERVAL_S)` (0.25 s) so nudge dispatch latency stays inside PERF-04's 500 ms budget without busy-spinning.
- `start_warmup_loop(stop_event: threading.Event) -> None` — main entry point. Spawned by `scheduler_service.main()`.
- `reset_state_for_tests() -> None` — clears module-level state. Only used by pytest fixtures.

per 15 min across scheduler cycles + on-app-open nudges combined.

  {"timestamp_iso": "2026-05-05T12:34:56+00:00",

   "sessid_changed": false}

"GET /personal/"). `success` is a boolean derived from outcome:

def start_warmup_loop(stop_event: threading.Event) -> None:
    """Daemon entry point. Spawned by scheduler_service.main()."""
    try:
        if stop_event.wait(BOOT_GRACE_S):
            return
        while not stop_event.is_set():
            cycle_start = time.monotonic()
            try:
                _run_cycle(stop_event)
            except Exception:  # noqa: BLE001
                logger.exception("keepalive: cycle crashed")
            deadline = cycle_start + CYCLE_INTERVAL_S
            _drain_nudges(stop_event, deadline)
    except Exception:  # noqa: BLE001 - never kill scheduler process
        logger.exception("keepalive: fatal thread exit")

        _LAST_WARMUP_AT.clear()
        CART_ADD_ACTIVE.clear()

        except queue.Empty:

**Implementation notes for the executor:**

- `_collect_linked_users()` lists `USER_COOKIES_DIR` for `{telegram_id}.json` filenames, reads `PHONE_MAP_PATH` for phone-mapped users, resolves each to its canonical cookies path via `bot.auth.get_user_cookies_path` + `backend.main._phone_cookies_path` patterns, and dedupes by resolved path. `user_id_hash = sha256(telegram_id or phone)[:12]`.
- `_pool_is_healthy()` MUST import `vless.manager` and `scheduler_service` lazily (inside the function, not at module top) to avoid circular-import issues — `scheduler_service` imports `keepalive.warmup` at top. Check `quarantined_count >= max(1, snap["size"] // 2)` — NOT strictly greater — per CONTEXT §Locked Defaults.
- Proxy manager singleton: reuse whatever `cart/vkusvill_api.VkusVillCart` accepts via `proxy_manager=` (see the existing `/api/cart/items` endpoint at `backend/main.py:3749` for the construction pattern). If no singleton helper exists, construct a minimal `VlessProxyManager(register_atexit=False)` once and cache under `_STATE_LOCK`.
- `_emit_event` uses `fcntl.flock` on POSIX for JSONL line atomicity (R2 in 62-CONTEXT.md). On Windows (local dev only), fall back to an `os.O_APPEND` open — kernel atomicity is sufficient for single-line writes. Never raises; wrap everything in try/except and log once via `logger.warning`.
- `_warmup_single_user` reads `CART_ADD_ACTIVE[user_id_hash]` under `_STATE_LOCK` BEFORE the HTTP request. If present AND its value < 3 s old (active cart-add), emit `outcome="cancelled_by_cart_add"` and return without hitting VkusVill. Check the flag again AFTER the request returns — if it appeared during the request, emit `outcome="raced"` instead of `"ok"`/`"http_error"`. Best-effort detection.
- Latency is `(time.monotonic() - start) * 1000` rounded to int ms. `sessid_changed = (old_sessid != new_sessid)` — both strings snapshotted before/after the warmup HTTP call (`httpx.get(WARMUP_URL, cookies=..., timeout=WARMUP_TIMEOUT, proxy=WARMUP_PROXY)`). `_LAST_WARMUP_AT[user_id_hash]` is updated regardless of outcome (prevents tight-retry spam).
- Warmup issues its own HTTP request rather than calling `cart._refresh_stale_session()` because the latter has a hardcoded `SESSID_REFRESH_TIMEOUT` (10s/10s) that blows PERF-05's 3s p95 budget. Parse the sessid from the response cookies with the same regex pattern as `cart/vkusvill_api.py:293-334`. To persist the updated cookies, either instantiate a `VkusVillCart(cookies_path=...)` and call `_persist_session_metadata`, or write the cookies JSON directly — executor picks the simpler path.
