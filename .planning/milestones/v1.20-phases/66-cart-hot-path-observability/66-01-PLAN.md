## Goal

Extend _build_reliability_snapshot() in backend/main.py with a cart_add sub-block computed from the in-memory _cart_add_attempts ledger. Extend the reasons[] list with cart_add_p95_high / cart_add_p95_critical per the thresholds locked in 66-CONTEXT.md D7.

## Contract

### Input
Ledger state at the moment of /api/health/deep:
- _cart_add_attempts: dict[str, dict] — each entry has created_at, resolved_at (optional), status, duration_ms (optional), user_id, product_id.
- Window cutoffs: now - 3600 s for 1 h metrics, now - 86400 s for 24 h metrics.

### Output
snapshot["cart_add"] is either:
- Omitted if zero attempts resolved within the last 3600 s, OR
- Present as:

```json
{
  "p50_ms": 1234,
  "p95_ms": 4500,
  "p99_ms": 9000,
  "success_rate_1h": 0.98,
  "success_rate_24h": 0.97,
  "double_add_rate_1h": 0.0,
  "window_sample_1h": 42,
  "window_sample_24h": 1024
}
```

Additionally, snapshot["reasons"] MAY gain:
- "cart_add_p95_high:{p95}ms" when 6000 < p95_ms <= 12000
- "cart_add_p95_critical:{p95}ms" when p95_ms > 12000

## Implementation

### New constants (near existing _DEEP_MAX_*_AGE_S)

```python
_HEALTH_CART_ADD_DEGRADED_P95_MS = 6000
_HEALTH_CART_ADD_UNHEALTHY_P95_MS = 12000
_HEALTH_CART_ADD_WINDOW_1H_S = 3600.0
_HEALTH_CART_ADD_WINDOW_24H_S = 86400.0
_HEALTH_CART_ADD_DOUBLE_ADD_WINDOW_S = 30.0
```

### New helper: _compute_cart_add_block

Returns a tuple (block_or_none, p95_reason_or_none). Uses _cart_add_attempts_lock only for the single list(...) snapshot; percentile computation happens outside the lock.

```python
def _compute_cart_add_block() -> tuple[dict | None, str | None]:
    """Compute the /api/health/deep cart_add block from _cart_add_attempts.

    Returns (block, p95_reason). block is None when no attempts resolved in
    the last hour. p95_reason is None when p95 is within the healthy budget.
    """
    import statistics
    now = _time.time()
    with _cart_add_attempts_lock:
        attempts = list(_cart_add_attempts.values())

    cutoff_1h = now - _HEALTH_CART_ADD_WINDOW_1H_S
    cutoff_24h = now - _HEALTH_CART_ADD_WINDOW_24H_S

    attempts_1h = [a for a in attempts if (a.get("resolved_at") or 0) >= cutoff_1h]
    attempts_24h = [a for a in attempts if (a.get("resolved_at") or 0) >= cutoff_24h]

    if not attempts_1h:
        return None, None

    durations = sorted(a["duration_ms"] for a in attempts_1h if a.get("duration_ms") is not None)
    if durations:
        if len(durations) == 1:
            p50 = p95 = p99 = durations[0]
        elif len(durations) == 2:
            p50 = durations[0]
            p95 = p99 = durations[-1]
        else:
            qs = statistics.quantiles(durations, n=100)
            p50 = int(qs[49])
            p95 = int(qs[94])
            p99 = int(qs[98])
    else:
        p50 = p95 = p99 = 0

    ok_1h = sum(1 for a in attempts_1h if a.get("status") == "success")
    ok_24h = sum(1 for a in attempts_24h if a.get("status") == "success")
    success_rate_1h = round(ok_1h / len(attempts_1h), 4) if attempts_1h else 0.0
    success_rate_24h = round(ok_24h / len(attempts_24h), 4) if attempts_24h else 0.0

    success_1h = [a for a in attempts_1h if a.get("status") == "success"]
    double_add = 0
    for i, a in enumerate(success_1h):
        for b in success_1h[i + 1:]:
            if (
                str(a.get("user_id")) == str(b.get("user_id"))
                and int(a.get("product_id", -1)) == int(b.get("product_id", -2))
                and abs((a.get("resolved_at") or 0) - (b.get("resolved_at") or 0)) <= _HEALTH_CART_ADD_DOUBLE_ADD_WINDOW_S
            ):
                double_add += 1
    double_add_rate_1h = round(double_add / len(success_1h), 4) if success_1h else 0.0

    block = {
        "p50_ms": int(p50),
        "p95_ms": int(p95),
        "p99_ms": int(p99),
        "success_rate_1h": success_rate_1h,
        "success_rate_24h": success_rate_24h,
        "double_add_rate_1h": double_add_rate_1h,
        "window_sample_1h": len(attempts_1h),
        "window_sample_24h": len(attempts_24h),
    }

    reason = None
    if p95 > _HEALTH_CART_ADD_UNHEALTHY_P95_MS:
        reason = f"cart_add_p95_critical:{int(p95)}ms"
    elif p95 > _HEALTH_CART_ADD_DEGRADED_P95_MS:
        reason = f"cart_add_p95_high:{int(p95)}ms"

    return block, reason
```

### _build_reliability_snapshot integration

After the existing reasons collection but before the severity classification:

```python
    # v1.20 OBS-04: cart-add latency and double-add visibility
    cart_add_block, cart_add_reason = _compute_cart_add_block()
    if cart_add_reason:
        reasons.append(cart_add_reason)
```

After snapshot = {...} is built, add:

```python
    if cart_add_block is not None:
        snapshot["cart_add"] = cart_add_block
```

The critical-reasons set stays unchanged ({"xray_bridge_not_listening", "no_cycle_state"}). cart_add_p95_critical is NOT a single-trigger critical — it only tips to unhealthy via the n_failed >= 3 OBS-02 gate. This is intentional: a p95 spike is evidence of a problem but not an outage.
