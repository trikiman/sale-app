## What Was Built

### `scripts/ablate_basket_add_payload.py`

CLI surface:
```
python scripts/ablate_basket_add_payload.py \
  --user-id USER --product-id PID --n-per-field N \
  [--output results.json] [--cookies-path PATH] [--dry-run]
```

Module-level `PAYLOAD_FIELDS` (a 16-tuple) is the single source of truth mirroring the dict literal in `cart/vkusvill_api.py::VkusVillCart.add()`. A docstring note above it instructs future maintainers to keep the two in sync; the dry-run's shape assertion is the mechanical guard that catches drift.

Core functions:

- `_build_payload(*, product_id, user_id, sessid, is_green=0, price_type=1)`: returns the exact 16-field dict (plus `sessid`) that `add()` sends.
- `_measure_one(cart_client, *, product_id, drop_field)`: live-mode only. Builds payload, optionally pops `drop_field`, POSTs via `cart._request(BASKET_ADD_URL, payload)`, times it, calls `cart.remove(...)` for cleanup. Returns `(success_bool, elapsed_ms)`. Exceptions are treated as failures so one network hiccup does not abort the whole sweep.
- `_summarize(samples)`: returns `{n, success_rate, p50_ms, p95_ms}`. Uses `statistics.quantiles(..., n=20)[18]` when `n >= 20`; falls back to rank-based nearest index for small-N (covers the dry-run with N=2).
- `_classify(baseline, per_field)`: returns `{droppable, required}` lists. Threshold `DROPPABLE_SUCCESS_THRESHOLD = 0.95`.
- `_run_dry(args)`: validates `_build_payload` output keys match `PAYLOAD_FIELDS` exactly (missing -> assertion error; extra -> assertion error) then emits synthetic results so the smoke check exercises the full JSON pipeline.
- `_run_live(args)`: authenticates via `VkusVillCart(cookies_path=...)`, iterates `PAYLOAD_FIELDS`, runs N cycles per field + N baseline cycles, classifies, writes JSON.

Output schema (stable across modes):
```json
{
  "mode": "dry_run|live",
  "product_id": 731,
  "user_id": "12345",
  "fields_ablated": ["id","xmlid","..."],
  "baseline": {"n": 20, "success_rate": 1.0, "p50_ms": 3600.0, "p95_ms": 4800.0},
  "per_field": {
    "id": {"n": 20, "success_rate": 0.10, "p50_ms": 3700.0, "p95_ms": 4900.0}
  },
  "verdict": {"droppable": ["coupon","cssInited"], "required": ["id","user_id","sessid"]}
}
```

Safety guards:
- `from cart.vkusvill_api import VkusVillCart` is lazy (inside `_run_live`) so the dry-run imports zero network-touching code.
- Cleanup `cart.remove(...)` in a try/except so a transient remove failure does not leave the sweep half-run.
- `--cookies-path` defaults to `data/browser_cookies_live.json` (the same file cart.vkusvill_api uses).

### `scripts/verify_v1.20.sh`

New Phase-64 section, inserted between the Phase-63 block and the cross-phase v1.19 regression gate:

- **64-A**: on EC2, clear `USE_FAST_CART_ADD_ENDPOINT` from the process env, `importlib.reload(cart.vkusvill_api)`, assert `USE_FAST_CART_ADD_ENDPOINT is False`. Catches any deploy that ships with the env var accidentally set in a systemd unit.
- **64-B**: on EC2, `from cart.vkusvill_api import FAST_CART_ADD_URL; assert FAST_CART_ADD_URL is None`. Catches a future deploy that populated the URL without updating the smoke gate (i.e. the spike landed but the smoke script was not re-evaluated).
- **64-C**: on EC2, `python3 scripts/ablate_basket_add_payload.py --dry-run --user-id 12345 --product-id 731 --n-per-field 2 >/dev/null`. Catches a deploy where the script has an import error or the CLI contract broke.

All three use the existing `_pass` / `_fail` / `ssh "$EC2_HOST"` helpers from the enclosing script.
