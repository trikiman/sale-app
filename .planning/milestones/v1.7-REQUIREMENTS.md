# Requirements: v1.7 Categories & Subgroups

**Milestone:** v1.7
**Created:** 2026-04-02
**Goal:** Add proper group/subgroup category structure across the app with favorite groups/subgroups and notifications.

## Data / Scraping

- [x] **DATA-01**: Category scraper stores `{group, subgroup}` for each product (not just flat category name)
- [x] **DATA-02**: Existing 16K products in `category_db.json` are re-scraped with subgroup data
- [x] **DATA-03**: Product catalog DB (`product_catalog` table) has group and subgroup columns populated from category_db

## Main Page UI

- [x] **UI-08**: Group category chips displayed on main page (existing behavior, preserved)
- [x] **UI-09**: Selecting a group shows a second row of subgroup chips for that group
- [x] **UI-10**: Subgroup row hidden when no group is selected
- [x] **UI-11**: Subgroup row hidden when the selected group has only 1 subgroup
- [x] **UI-12**: User can tap heart on a group/subgroup chip to favorite the group/subgroup itself
- [x] **UI-13**: Products correctly filtered by selected group and subgroup

## History Page

- [x] **HIST-01**: History page shows group filter chips (category groups)
- [x] **HIST-02**: Selecting a group on history page shows subgroup chips (same drill-down UX as main page)
- [x] **HIST-03**: Same subgroup hide rules apply (no group selected → no subgroups, 1 subgroup → no row)
- [x] **HIST-04**: Group/subgroup favorite works on history page too

## Favorites

- [x] **FAV-03**: Group/subgroup favorites stored per-user in DB (separate from product favorites)
- [x] **FAV-04**: Favorites view shows group/subgroup favorites as single entries alongside individual product favorites

## Notifications

- [x] **BOT-06**: Telegram notifier checks group/subgroup favorites — when any product in a favorited group/subgroup goes on sale, the user gets notified

## Outcome Notes

- Search completeness for non-sale products stayed out of scope and remains a candidate for the next milestone.
- Multi-subgroup fidelity is preserved in `category_db.json`, but `product_catalog` still stores one subgroup field per product.
- Notification matching was verified with automated regression coverage and an EC2 dry run.
