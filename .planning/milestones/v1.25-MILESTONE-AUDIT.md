## REL-19 Hotfix (Session-Unplanned)

Not in REQUIREMENTS.md but shipped tonight because of a production incident.

**Incident:** 2026-05-13 00:04 → 01:13 MSK, 69-minute family-facing outage. User reported "`Обновлено: 00:04 now 1 13 why AGIN outdated`". Root cause: v1.24 Phase 77 REL-19 graceful-degrade skipped not just expensive Chrome startup but also the `ensure_pool()` recovery call. With no path calling pool refresh, the pool stayed at 0 forever.

**Fix:** `b65cde7` — `_run_scraper_set` graceful-degrade block now ALWAYS calls `ensure_pool()` before deciding skip-vs-run. If pool recovers mid-cycle → falls through to normal scrape + resets counter + emits `scheduler_pool_recovered` event. If still dead after refresh → honors skip as before.

**Pinned by:** `test_collapse_replay_event2_dead_cycles_trigger_recovery_attempts` (Phase 81 QA-06). Any future refactor that re-introduces the skip-without-recovery pattern will fail this test.

**Live recovery:** Pool went 0 → 10 within seconds of manually triggering `ensure_pool()` on EC2 after deploy.
