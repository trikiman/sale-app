# Plan 74-01 — Remove Probe Loop + Tighten Timeouts

## Goal

Replace the two-phase probe loop in `backend/main.py::product_details` with a single-path fetch through the xray bridge.

## Changes

### `backend/main.py::product_details` (lines ~526-635)

**Before** (structure):
```python
# Phase 1: HEAD probe to find live proxy (4s timeout × N proxies)
dead_proxies = []
working_proxy = None
for entry in pool:
    ...head(timeout=4s)...
# Phase 2: fetch only if probe succeeded
fetch_attempts = 3 if working_proxy else 0
for attempt in range(fetch_attempts):
    ...get(timeout=connect=4, read=6)...
```

**After**:
```python
# Single path: fetch via xray bridge with retries
proxy_url = "socks5://127.0.0.1:10808"
retry_count = 0
html = None
last_error = None
for attempt in range(3):
    retry_count = attempt + 1
    try:
        async with httpx.AsyncClient(
            timeout=httpx.Timeout(connect=1.0, read=3.0, write=1.0, pool=1.0),
            follow_redirects=True, proxy=proxy_url,
        ) as client:
            resp = await client.get(url, headers={
                "User-Agent": ua,
                "Accept": "text/html,application/xhtml+xml",
            })
        if resp.status_code == 200 and len(resp.text) > 500:
            html = resp.text
            logger.info(
                f"[DETAIL-FETCH] Product {product_id}: {len(html)}b via bridge "
                f"on attempt {retry_count} in {resp.elapsed.total_seconds():.2f}s"
            )
            break
        logger.warning(
            f"[DETAIL-FETCH] Product {product_id}: bridge returned status "
            f"{resp.status_code} on attempt {retry_count}"
        )
        last_error = f"status_{resp.status_code}"
    except (httpx.ConnectError, httpx.TimeoutException) as e:
        last_error = type(e).__name__
        logger.warning(
            f"[DETAIL-FETCH] Product {product_id}: bridge {type(e).__name__} "
            f"on attempt {retry_count}"
        )
    except Exception as e:
        last_error = type(e).__name__
        logger.warning(
            f"[DETAIL-FETCH] Product {product_id}: bridge error "
            f"on attempt {retry_count}: {e}"
        )
```

### Constants

Add near other timeout constants (top of file):
```python
_DETAIL_BRIDGE_PROXY = "socks5://127.0.0.1:10808"
_DETAIL_TIMEOUT = httpx.Timeout(connect=1.0, read=3.0, write=1.0, pool=1.0)
_DETAIL_MAX_RETRIES = 3
```

Keep them module-level so the unit test in 74-02 can monkey-patch for fast failure simulation.

### Logging Tag Change

`[DETAIL-PROXY]` → `[DETAIL-FETCH]` — the word "proxy" implied a multi-proxy probe that no longer exists. This matches the post-v1.15 terminology used in `vless/preflight.py` and keepalive logs.

### Preserved Behavior

- Cache check via `detail_service.read_cache(product_id)` unchanged.
- Fallback path unchanged: `html is None` or `len(html) < 500` → `_fallback_product_details`.
- HTML parsing + `detail_service.write_cache` unchanged.
- 404 on unknown product_id unchanged.
