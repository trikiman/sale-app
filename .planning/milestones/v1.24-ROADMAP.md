## v1.24 Pool Self-Heal Hardening + Outage UX (ACTIVE — started 2026-05-13)

Eliminate the ~1h family-facing outage observed 2026-05-13 when VLESS pool collapsed. Recovery must take minutes not an hour. During rebuild, users see cached data with stale badges instead of empty "0 всего" grid. Codify style guide v2 rules into automated lint checks.

Driving evidence:

| Area | Issue | Measured | v1.24 Phase |
|---|---|---|---|
| Pool recovery time | ~1h for pool to hit `min_healthy=7` after collapse | 16:19 → 17:30 MSK 2026-05-13 | Phase 77 (REL-16/17/18/19) |
| Empty grid during outage | `/api/products` strips all products when all sources stale | Observed 2026-05-13 — 174 cached items hidden | Phase 78 (UX-STALE-01/02) |
| Style drift regressions | Header visual-weight violation, off-scale paddings | Style guide v2 lacks automated enforcement | Phase 79 (TOOL-02/03) |

**Goal:** Pool recovery p95 ≤ 10 min. Zero "empty grid" when cached data exists. Style guide v2 rules enforced via CI.
**Granularity:** Small
**Phases:** 3 (77-79)
**Requirements:** 9 (4 REL, 2 UX, 2 TOOL, 1 optional OBS)

### Phases

- [ ] **Phase 77: Pool Self-Heal Hardening** — Persistent quarantine deadlist (20-min TTL) in `data/pool_quarantine.json`; refresh throttle (`REFRESH_MIN_INTERVAL_S = 60`); lower water mark (`min_healthy: 10` + rate-of-decline check `3+ nodes in 5 min`); scheduler graceful degrade when pool 0 for >2 min (skip cycle with `scheduler_pool_dead` JSONL event instead of exit 1). Optionally emit Telegram admin alert if pool stays 0 > 10 min. Smoke gate: simulate pool death on EC2, assert recovery <10 min via `/api/health/deep` polling. (REL-16/17/18/19, optional OBS-08)

- [ ] **Phase 78: Stale-State UX** — Backend `/api/products` returns last-good snapshot with `stale_all: true` flag instead of stripping products when all 3 sources stale. Frontend renders cached products with `⏳ stale` per-card badge. Stale banner upgraded from thin line to prominent bordered card per style guide v2 "State Patterns > Stale" section. Live MCP: simulate all-stale state via scheduler pause, assert grid shows cached products. (UX-STALE-01/02)

- [ ] **Phase 79: Style Guide v2 Enforcement** — `stylelint` config rejects off-scale spacing values; `eslint` config rejects inline `style=` in JSX. Both wired into `npm run lint` + pre-commit. Baseline existing violations as eslint-disable comments with `TODO(v1.25)` markers. Smoke gate: `npm run lint` green, committed `TODO` list documents remaining debt. (TOOL-02/03)

### Phase Details

### Phase 77: Pool Self-Heal Hardening
**Goal:** VLESS pool recovers from full collapse (20 quarantined, 0 healthy) in ≤10 min instead of ~1h.
**Depends on:** v1.23 closed.
**Requirements:** REL-16/17/18/19, optional OBS-08 — plus continued OPS-21/22/23.
**Success Criteria:**
  1. [ ] `data/pool_quarantine.json` persists dead-node set with TTL 20 min. Refresh skips nodes in quarantine.
  2. [ ] `REFRESH_MIN_INTERVAL_S = 60` throttle in `vless/manager.py::ensure_pool`. Second refresh within 60s is a no-op.
  3. [ ] `min_healthy = 10` (was 7). Rate-of-decline check triggers refresh if pool drops by 3+ in 5 min.
  4. [ ] `scheduler_service.py` skips scrape cycle with exit 0 + `scheduler_pool_dead` JSONL event when pool 0 for >2 min.
  5. [ ] Unit test: `tests/test_vless_quarantine.py` covers TTL expiry, refresh skip, throttle no-op.
  6. [ ] Live EC2 smoke: force pool death, measure `/api/health/deep` pool size recovery curve. Assert p95 ≤ 10 min.
  7. [ ] v1.23 + earlier regression green.
**Plans:** TBD via `/gsd-plan-phase 77`

### Phase 78: Stale-State UX
**Goal:** Zero "empty grid" when `proposals.json` has cached products, regardless of source staleness.
**Depends on:** Phase 77 not required (independent UX fix), but pairs naturally — REL-19 emits `scheduler_pool_dead` which backend can surface.
**Requirements:** UX-STALE-01/02 — plus continued OPS-21/22/23.
**Success Criteria:**
  1. [ ] `/api/products` returns last-good snapshot with `stale_all: true` when all 3 sources stale. Legacy v1.22 Phase 66.1 phantom-strip behavior removed for the all-stale case (single-color-stale strip preserved).
  2. [ ] Per-card `⏳ stale` badge renders on cached products when source is stale.
  3. [ ] Stale banner upgraded per style guide v2 (prominent card, not thin line). Visible above the fold on mobile (360px).
  4. [ ] Estimated recovery time calculated from last scheduler success + typical cycle duration (~3 min). Shown in banner.
  5. [ ] Live MCP: pause scheduler on EC2, wait 16 min (past stale threshold), open MiniApp, assert grid shows cached products + prominent banner.
  6. [ ] v1.23 + earlier regression green.
**Plans:** TBD via `/gsd-plan-phase 78`

### Phase 79: Style Guide v2 Enforcement
**Goal:** Automated CI enforcement of style guide v2 rules — spacing scale, no inline styles.
**Depends on:** nothing (tooling-layer, orthogonal to pool/UX).
**Requirements:** TOOL-02/03 — plus continued OPS-21/22/23.
