# Plan 02: Cart Regression Matrix

## Objective

Lock the new cart contract down with focused backend regression tests so future changes cannot silently break the 5-second UX path or the quantity flow.

## must_haves

- [ ] Backend tests cover immediate success, pending→success, pending→failure, and legacy timeout compatibility
- [ ] Quantity route and decimal quantity serialization stay covered
- [ ] Admin diagnostics payload is covered by route tests
- [ ] One repeatable regression command exists for the cart flow

<action>
Extend the existing cart backend tests so Phase 45 has explicit regression coverage:
