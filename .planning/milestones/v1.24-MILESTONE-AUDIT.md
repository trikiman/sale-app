## Executive Summary

**9/9 requirements shipped across 3 phases in a single day.** v1.24 was the direct response to the ~1h family-facing VLESS pool outage observed during v1.23 verification session — pool recovery time is now bounded by (a) persistent quarantine memory so dead nodes aren't re-probed, (b) a 60s refresh throttle that prevents the 19-refreshes-in-15-min thrash pattern, (c) earlier low-water-mark warning (MIN_HEALTHY 7→10 + rate-of-decline check), (d) scheduler graceful degrade when pool 0 for 2+ cycles. During the recovery window, users now see cached products with `⏳` stale badges + a prominent bordered banner instead of the "0 всего" empty grid. Style guide v2 enforcement tooling (stylelint + eslint forbid-dom-props) baselined 46 inline-style debt entries at WARN level for v1.25 cleanup.

## Phase-by-Phase

### Phase 77 — Pool Self-Heal Hardening (REL-16/17/18/19)

**Commits:** `e188e6c`, `d44d410`, `a0aa6dc` (3 atomic per plan discipline)

**Scope:**
- New module `vless/quarantine.py` (~180 LOC) — persistent JSON deadlist with 20-min TTL default, 4h TTL for repeat offenders (fail_count ≥3), atomic writes via `.tmp` + `os.replace`, I/O failures swallowed at DEBUG level
- `vless/manager.py`:
  - `MIN_HEALTHY` 7 → 10
  - Added `REFRESH_MIN_INTERVAL_S = 60.0`, `RATE_DECLINE_WINDOW_S = 300`, `RATE_DECLINE_THRESHOLD = 3`, `_POOL_HISTORY_MAX = 20`
  - Instance attrs `_last_refresh_monotonic`, `_pool_size_history` (deque)
  - Static method `_monotonic()` for test-patchable clock
  - `ensure_pool` rewritten: record size sample → decline check → throttle check → refresh only if warranted
  - `refresh_proxy_list` reads `quarantine.get_quarantined_hosts()`, excludes candidates, records probe failures via `record_probe_failures`
- `scheduler_service.py`:
  - New `_SCHEDULER_EVENTS_PATH`, `_emit_scheduler_event`, `_is_pool_dead` helpers
  - `_run_scraper_set` gains graceful-degrade guard: if pool dead 2+ cycles, skip scrape + emit `scheduler_pool_dead` event

**Tests:** `tests/test_vless_quarantine.py` — 15/15 passed (10 quarantine + 2 ensure_pool + 3 scheduler).
