**Part B (stashed, broken)**: `vless/manager.py` `_probe_candidates_in_parallel` calls a `_tcp_prefilter_candidates(candidates)` helper that **does not exist**. Need to write it. Sketch:

```python
def _tcp_prefilter_candidates(self, candidates: list[VlessNode]) -> list[VlessNode]:
    """Cheap TCP open() check before spinning up xray subprocesses.
    Reject candidates where host:port doesn't open in 2s.
    Stamp rejected nodes with extra['rejected_reason'] = 'tcp_unreachable'
    so they get the 60s soft-tier quarantine via the existing classifier.
    """
    import socket, concurrent.futures
    survivors = []
    def _check(node):
        try:
            with socket.create_connection((node.host, node.port), timeout=2.0):
                return node
        except (OSError, socket.timeout):
            node.extra["rejected_reason"] = "tcp_unreachable"
            return None
    with concurrent.futures.ThreadPoolExecutor(max_workers=16) as pool:
        for r in pool.map(_check, candidates):
            if r is not None:
                survivors.append(r)
    rejected = len(candidates) - len(survivors)
    if rejected:
        self._log(
            f"TCP pre-filter: {len(survivors)}/{len(candidates)} candidates reachable "
            f"({rejected} dead on TCP layer)"
        )
    # IMPORTANT: also feed the rejected set into the quarantine ledger
    # so they don't get re-probed every cycle. They're tcp-dead, soft-tier
    # quarantine (60s) is enough — they recover quickly if they come back up.
    if rejected:
        from vless import quarantine
        dead_keys = [
            f"{c.host}:{c.port}" for c in candidates
            if c not in survivors
        ]
        quarantine.record_probe_failures(dead_keys, reason="tcp_unreachable")
    return survivors
```

The reason classifier in `manager.py` already maps `'timeout'` substring → `probe_timeout` (60s soft-tier), but `tcp_unreachable` would currently fall through to generic `probe_error`. Either rename the reason to `probe_timeout` here, OR add `tcp_unreachable` to the classifier in `_probe_candidates_in_parallel`. Either's fine.
